# <a name="builtin-report-reference"></a>組み込みレポートの参照
最初は、組み込みレポートを基礎的なテンプレートとして使用してカスタム レポートを作成することをお勧めします。 次の表は、監視に使用する組み込みレポートおよびイベントの種類のリストです。  
  
## <a name="builtin-report-list"></a>組み込みレポートのリスト  
  
|レポートの種類|組み込みレポートの名前|説明|  
|-----------------|---------------------------|-----------------|  
|セキュリティ|場所ごとのアクティビティ|このレポートには、クラウド アプリのアクティビティの実行元の国、およびイベント数やユーザー数などの各国のアクティビティのボリュームを表すさまざまなパラメーターのリストが表示されます。ユーザーの地理的分散を大まかに把握する際に使用します。|  
|セキュリティ|ユーザーごとのアクティビティ ソース|複数の場所から、または過度な数の IP アドレスを使用してクラウド環境でアクティビティを実行したユーザー。 このような場合、ユーザーが出張中であるか、または資格情報が悪用された可能性があります。|  
|セキュリティ|ブラウザーの使用|閲覧をきっかけとする攻撃は、攻撃方法として最も一般的です。 ベンダーはブラウザー ソフトウェアのセキュリティ保護に膨大なリソースを投資し、更新をエンドポイントに配布するための効果的な更新メカニズムを作成しています。 長期間更新されていない非推奨のブラウザーを使用すると、一般に入手可能な攻撃キットを使用して悪意のある人物が容易に攻撃できるようになります。 このレポートには、過去 30 日間にユーザーがクラウド サービスにアクセスする際に使用したブラウザーのリストが表示されます。|  
|セキュリティ|IP アドレス|このレポートには、Cloud App Security で保護されているクラウド環境でアクティビティを実行したデバイスが使用した IP アドレスのリストが表示されます。 このレポートは、Cloud App Security が蓄積した監査ログに基づいています。 IP アドレスは通常、地理的な場所および送信元の組織に関連付けられています。 このレポートを使用すると、保護されたサービスへの接続に使用された不審な IP アドレスを識別できます。 IP アドレスをクリックすると、各 IP アドレスの監査ログを表示できます。|  
|セキュリティ|IP アドレス - 特権アカウント|このレポートには、Cloud App Security で保護されているクラウド環境で管理アクティビティを実行したデバイスが使用した IP アドレスのリストが表示されます。 このレポートは、Cloud App Security が蓄積した監査ログに基づいています。 IP アドレスは通常、地理的な場所および送信元の組織に関連付けられています。 このレポートを使用すると、保護されたサービスへの接続に使用された不審な IP アドレスを識別できます。 IP アドレスをクリックすると、各 IP アドレスの監査ログを表示できます。|  
|セキュリティ|OS の使用状況|オペレーティング システム (OS) のセキュリティ上の欠陥は、とても危険な脆弱性になることがよくあります。 ベンダーは OS のセキュリティ保護に膨大なリソースを投資し、更新をエンドポイントに配布するための効果的な更新メカニズムを作成しています。 しかし、サポート期間が終了した非推奨の OS にはこのような更新メカニズムが含まれていないため、一般に入手可能な攻撃キットを使用して悪意のある人物が容易に攻撃できるようになります。 このレポートには、過去 30 日間にユーザーがクラウド サービスにアクセスする際に使用した OS のリストが表示されます。|  
|セキュリティ|完全にリモートなユーザー|次のユーザーは、社内の他の従業員と IP アドレスをまったく共有していません。 これは、リモートのチームや従業員、委託業者、外部サポート サービスに見られる特性です。 ただし、攻撃者が設置したバックドア ユーザーであることも考えられます。|  
|ユーザー管理|クラウド アプリの概要|このレポートでは、クラウド アプリ、およびファイル数やユーザー数などの各アプリのアクティビティのボリュームを表すさまざまなパラメーターのリストが表示されます。各アプリの概要、およびユーザーの各アプリの使用状況を把握する際に使用します。|  
|ユーザー管理|非アクティブなアカウント|非アクティブなアカウントとは、クラウド インスタンスにアクセス可能なユーザー アカウントのうち、過去 60 日間まったくイベントを実行していないものを指します。 このアカウントは既にアクティブでないと考えられるため、今後悪意のある人物や退職した従業員が使用することを防止するために、一時停止することをお勧めします。 このベスト プラクティスは、セキュリティ体制を強化するだけではなく、運用コストの削減にもつながります。|  
|ユーザー管理|特権ユーザー|このレポートには、企業サービスの権限が管理者などに昇格されたユーザーのリストが表示されます。 このような特権アカウントからは、企業の情報やネットワーク構成に高い権限でアクセスできるため、悪意のある人物の攻撃手段として使用される場合があります。 過去 1 か月間使用されていない特権アカウントは連続的なデータ漏えいの原因になる可能性があり、企業の IT セキュリティ意識が不足していると言えます。 監査ログで昇格されたユーザー特権を詳細に調査し、不要な権限は取り消すことを検討します。|  
|ユーザー管理|特別な特権アカウント|Salesforce には、[Modify all data (すべてのデータを変更)]、[View all data (すべてのデータを表示)]、[Manage all users (すべてのユーザーを管理)] などの複数の種類の特権アカウントがあります。 このような特権アカウントからは、企業の情報や構成に高い権限でアクセスできるため、悪意のある人物から攻撃手段として使用される場合があります。|  
|ユーザー管理|ユーザー ログオン|このレポートには、サービス別およびユーザー別のログオンの試行回数と成功回数、ログオンに失敗した割合が詳細に表示されます。 ログオン試行の失敗の割合が高い場合は、ある時点でユーザー アカウントが攻撃を受けた可能性があります。|  
|データ管理|データ保持期間|データ保持ポリシーは、所有している情報を事前に定義された期間保持することや情報をアーカイブする手順、保持期限を経過した情報を破壊するガイドライン、訴訟中の情報を処理する特殊なメカニズムなどに関する問題を扱います。 データ保持期間は複雑なバランスに基いて決定されます。 一方では法令で保持を義務付けられている情報やビジネス上不可欠な情報を保存する必要があります。 これに対して、無関係で古い非生産的なデータは可能な限り早く削除することが推奨されます。 クラウドでデータ ストレージを使用する場合は、さらに問題が複雑になります。 何年も前の、データのハード コピーを保存していた時代と同じように、クラウド データも法令やビジネス、個人的な要件に基づいて一定期間保持する必要があります。 クラウド環境でデータ保持ポリシーを強制適用するもう 1 つのメリットは、ストレージ領域を解放しクラウドの運用コストを削減できることです。 次のレポートは、クラウド環境に保存されているデータの可視化に貢献します。これは、既存のデータ保持ポリシーをお客様の企業のクラウド環境に拡張する上で非常に重要な手順です。|  
|データ管理|データ共有の概要|このレポートには、クラウド サービスに保存されているファイルの数がアクセス許可ごとに表示されます。 クラウド サービスは、アクセスが容易でユビキタスのため、非常に簡単に共有を作成できます。<br /><br /> ファイルの所有者以外のユーザーと共有されていないファイルは、プライベート ファイルと呼ばれます。 ファイルが共有されている場合、Cloud App Security は 4 種類の状態に区別します。<br /><br /> 公開共有 (Web) ファイルは、認証を受けなくても検索エンジンの結果などからアクセスできるファイルです。<br /><br /> 公開共有ファイルは、認証を受けなくてもリンクからアクセスできるファイルです。<br /><br /> 外部共有ファイルは、クラウド サービスの認証を受けると社外のユーザーがアクセスできるファイルです。<br /><br /> 内部共有ファイルは、社内のすべてまたは一部のユーザーがアクセスできるファイルです。|  
|データ管理|ファイル拡張子|このレポートには、クラウド サービスに保存されているファイルの拡張子のリストが表示されます。 クラウド内のファイルの種類とその共有状況を大まかに把握する際に使用します。|  
|データ管理|ドメイン別の受信共有|このレポートには、従業員とファイルが共有されているドメインのリストが表示されます。 このレポートでは、ファイルを共有している共同作業者、共有されているファイル数、企業ユーザーのファイルが共有されている相手をドメインごとに詳細に確認できます。 外部との共同作業の受け入れは受動的であるため、従業員にポリシーが強制適用されることはありません。 ただし、企業が所有しているはずのファイルが実際には外部ユーザーに所有されていて、共有を通じてアクセスされたり、最悪の場合共有ファイルを通じて従業員に対するフィッシング攻撃が実行される場合があります。|  
|データ管理|孤立ファイル|このレポートには、一時停止中でありながら Cloud App Security で保護されているクラウド サービスのいずれかでまだファイルの所有者となっているユーザーのリストが表示されます。 これらのファイルがコンプライアンスやポリシーに違反している場合、管理されておらず行方不明になっています。 削除するか、またはアクティブ ユーザーに所有権を移動することをお勧めします。|  
|データ管理|ドメイン別の送信共有|このレポートには、従業員が企業ファイルの共有に使用しているドメインのリストが表示されます。 レポートには、ドメインごとに、そのドメインを使用してファイルを共有している企業ユーザー、共有しているファイル、ファイルの共有相手の共同作業者が含まれます。 これらのドメインを使用した共有は、関連する各サービスのページの [ファイル] タブで管理することをお勧めします。|  
|データ管理|共有ファイルの所有者|このレポートには、外部と企業のファイルを共有しているユーザーのリストが表示されます。 外部共有ファイルは、特定の外部の共同作業者と共有されています。 公開共有ファイルは、プライベート リンクからアクセスできるインターネット ユーザー全員に公開されています。ただし、明示的に示されているリンクからのみアクセスできます。 公開共有ファイル (インターネット) はインターネット ユーザー全員に公開されていて、検索エンジンの結果からもアクセスできます。  過度な量のファイルを共有しているユーザーを発見した場合、[ファイル] タブを使用して過度な共有アクセス許可の内容を調査し、ユーザーに連絡して外部共有の使用状況の詳細について確認することをお勧めします。|  
|データ管理|個人ユーザー アカウント|このレポートには、Cloud App Security で保護されているクラウド サービス内のファイルへのアクセスに使用された外部ユーザー アカウントのうち、個人ユーザー アカウントと推測されるもののリストが表示されます。 [企業ユーザー] 列に、各個人アカウントの所有者と推測されるユーザーが表示されます。 この組み合わせは、弊社独自のアルゴリズムを使用して計算されます。 企業データが従業員の個人アカウントで共有されると、機密情報が以下のような脅威にさらされる恐れがあります。<br /><br /> (1) 個人アカウントは、セキュリティおよび IT チームの手でプロビジョニングされていません。 ユーザーが気付かないうちに危険な状態に陥る可能性があり、また企業のセキュリティ ポリシー (パスワードのリセットなど) が適用されません。<br /><br /> (2) 個人アカウントで企業データを共有する場合、外部の共同作業者と共有する場合と比較して注意力が低くなりがちです。<br /><br /> (3) 内部の共同作業者による共有は、ソーシャル エンジニア リング攻撃である可能性があります。 攻撃者は、任意の従業員の名前に似た名前のアカウント (john.doe@contoso.com や jdoe12@gmail.com) など) を使用して機密データへのアクセス許可を要求し、権限を取得します。 内部の共同作業者は、気付かないうちに攻撃者のアカウントを使用して企業データを共有する場合があります。 個人アカウントの使用は、Cloud App Security では推奨されていません。|  
|データ管理|機密ファイル名|このレポートには、ファイル名のヒューリスティック スキャンに基づき、共有アクセス許可が内容に適合していないため注意が必要と判断されたファイルのリストが表示されます。 このヒューリスティック スキャンは、ファイルの内容全体をスキャンしなくても機密性の高いファイルを識別できるため、有用なものです。 レポート内の各ファイルはスキャンにより機密性が高いと判断されたもので、本来は [カテゴリ] 列に詳細に示されたカテゴリに属していると考えられます。|
