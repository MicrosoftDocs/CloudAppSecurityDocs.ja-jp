# <a name="manage-alerts"></a>アラートの管理

*適用対象:Microsoft Cloud App Security*

この記事では、Cloud App Security ポータルで発生したアラートの操作方法について説明します。

> [!NOTE]
> アラートはそれぞれのポリシーで管理され、電子メール、テキスト メッセージ、またはその両方として送信されるように構成できます。

## <a name="manage-your-alerts"></a>アラートを管理する

アラートは、クラウド環境をより深く理解するためのエントリ ポイントです。 検出された内容に基づいて新しいポリシーを作成することが必要になる場合があります。 たとえば、管理者がグリーンランドからサインインしていて、組織内のどのユーザーも以前にグリーンランドからサインインしたことがないとします。 管理者アカウントがその場所からのサインインに使用されると自動的に中断するポリシーを作成できます。

すべてのアラートを確認し、ポリシーを変更するためのツールとして使用することをお勧めします。 無害なイベントが既存のポリシーに違反していると見なされる場合は、不要なアラートが減少するようにポリシーを調整します。

1. **[アラート]** ページの **[解決状態]** で、 **[未解決]** を選択します。

    ダッシュボードのこのセクションでは、確立されたポリシーの不審なアクティビティまたは違反が完全に可視化されます。 これは、クラウド環境用に定義したセキュリティ体制を保護するのに役立ちます。

    ![アラート](media/alerts.png "アラート")

2. 各アラートについて、違反の性質と必要な応答を調査して判断する必要があります。

    - アラートの種類または重要度別にアラートをフィルター処理して、最も重要なものを最初に処理することができます。

    - 特定のアラートを選択します。 アラートの種類に応じて、アラートを解決する前に実行できるさまざまなアクションが表示されます。

    - アプリに基づいてフィルター処理できます。一覧に表示されているアプリは、Cloud App Security によってアクティビティが検出されたアプリです。

    - アラートを調査する際には、次の 3 種類の違反を処理する必要があります。

       - **重大な違反** - 重大な違反には迅速な対応が必要です。  
     *例:*
         - 不審なアクティビティのアラートの場合は、ユーザーがパスワードを変更するまでアカウントを中断できます。
         - データ リークが発生した場合は、アクセス許可を制限したり、ファイルを検査したりできます。
         - 新しいアプリが検出された場合は、プロキシまたはファイアウォールでサービスへのアクセスをブロックできます。

       - **疑わしい違反** - 問題のある違反は、さらに調査が必要です。
         - アクティビティの性質について、ユーザーまたはユーザーのマネージャーに連絡できます。
         - 詳細情報が表示されるまで、アクティビティは開いたままにしておきます。

       - **承認された違反または異常な動作** - 承認された違反または異常な動作は、正当な使用によって発生する可能性があります。
         - アラートは無視してかまいません。

3. アラートを無視する場合はいつでも、アラートを無視する理由に関するフィードバックを送信することが重要です。 Cloud App Security チームは、このフィードバックをアラートの精度を示すものとして使用します。 その後、この情報は、今後のアラートのために機械学習モデルを微調整するために使用されます。 次のガイドラインに従って、アラートを分類する方法を決定できます。
    - 正当な使用によってアラートがトリガーされ、それがセキュリティ上の問題ではない場合は、次のいずれかの種類になります。

      - 無害な陽性:アラートは正確ですが、アクティビティは正当です。 アラートを無視し、その理由を **[実際の重要度は低いです]** または **[興味なし]** に設定できます。
      - 偽陽性:アラートが不正確です。 アラートを無視し、理由を **[アラートが正確ではありません]** に設定します。
    - ノイズが多すぎてアラートの正当性と精度を判断できない場合は、アラートを無視し、理由を **[類似のアラートが多すぎます]** に設定します。
    - 真陽性:アラートが、内部関係者または部外者によって悪意をもってまたは意図せずにコミットされた実際の危険なイベントに関連している場合は、イベントを修復するための適切なアクションをすべて実行した後に、イベントを **[解決]** に設定する必要があります。

## <a name="alert-types"></a>アラートの種類

次の表に、トリガーできるアラートの種類と推奨される解決方法を示します。

|アラートの種類|[説明]|推奨される解決方法|
|----------------|-----------------|----------------------------|
|アクティビティ ポリシー違反|この種類のアラートは、作成したポリシーによって発生します。|この種類のアラートを一括で処理するには、ポリシー センターで作業してそれらを軽減することをお勧めします。<br /><br /> フィルターを追加してより細かく制御することで、不要なエンティティを除外するようにポリシーを微調整します。<br /><br />ポリシーが正確で、アラートが正当化されていて、違反をすぐに停止する場合は、ポリシーに自動修復を追加することを検討してください。|
|ファイル ポリシー違反|この種類のアラートは、作成したポリシーによって発生します。| この種類のアラートを一括で処理するには、ポリシー センターで作業してそれらを軽減することをお勧めします。<br /><br /> フィルターを追加してより細かく制御することで、不要なエンティティを除外するようにポリシーを微調整します。<br /><br />ポリシーが正確で、アラートが正当化されていて、違反をすぐに停止する場合は、ポリシーに自動修復を追加することを検討してください。|
|危険な状態のアカウント|この種類のアラートは、セキュリティ侵害を受けたアカウントが Cloud App Security によって識別された場合にトリガーされます。 これは、承認されていない方法でアカウントが使用された可能性が非常に高いことを意味します。|ユーザーに連絡し、パスワードを変更したことを確認するまで、アカウントを中断することをお勧めします。|
|非アクティブなアカウント|このアラートは、接続されているいずれかのクラウド アプリで過去 60 日間にアカウントが使用されていない場合にトリガーされます。|ユーザーとユーザーの管理者に連絡して、アカウントがまだアクティブであるかどうかを確認します。 アクティブでない場合は、ユーザーを中断し、アプリのライセンスを終了します。|
|新しい管理者ユーザー|接続されているアプリの特権アカウントが変更されたことを警告します。|ユーザーに対して、新しい管理者のアクセス許可が実際に必要であることを確認します。 必要でない場合は、管理者特権を失効させて露出を減らすことをお勧めします。|
|新しい管理者の場所|接続されているアプリの特権アカウントが変更されたことを警告します。|この異常な場所からのサインインが正当であることを確認します。 正当でない場合は、管理者アクセス許可を失効させるか、アカウントを中断して露出を減らすことをお勧めします。|
|新しい場所|新しい場所からの接続されているアプリへのアクセスに関する情報通知アラート。このアラートは、国ごとに 1 回だけトリガーされます。|特定のユーザーのアクティビティを調査します。|
|検出された新しいサービス|このアラートは、シャドウ IT に関するアラートです。 Cloud Discovery によって新しいアプリが検出されました。|<ul><li>アプリ カタログに基づいてサービスのリスクを評価します。</li><li>アクティビティを詳細表示して、使用パターンと普及率を把握します。</li><li>アプリを承認するか、未承認にするかを決定します。</li><br /></ul>承認されていないアプリの場合:<br /><br /><ul><li>プロキシまたはファイアウォールでの使用をブロックできます。</li><li>承認されていないアプリと承認されたアプリが同じカテゴリ内にある場合は、承認されていないアプリのユーザーの一覧をエクスポートできます。 次に、それらのユーザーに連絡して、それらを承認されたアプリに移行します。</li></ul></li>|
|疑わしいアクティビティ|このアラートでは、組織内の予期されるアクティビティまたはユーザーとは一致しない異常なアクティビティが検出されたことが通知されます。|動作を調査し、ユーザーに確認します。<br /><br />この種類のアラートは、環境に関する詳細を理解し、これらのアラートを使用して新しいポリシーを作成し始めるのに最適です。 たとえば、接続されているいずれかのアプリに大量のデータが突然アップロードされた場合、その種類の異常な動作を制御するルールを設定できます。|
|個人アカウントの使用|このアラートでは、接続されているアプリのリソースに新しい個人アカウントがアクセスできることが通知されます。|外部アカウントでユーザーのコラボレーションを削除します。|
