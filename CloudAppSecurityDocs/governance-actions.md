# <a name="governing-connected-apps"></a>接続されているアプリを管理する

*適用対象: Microsoft Cloud App Security*

ガバナンスによって、アプリ全体におけるユーザーの作業をリアルタイムで制御できるようになります。 接続されているアプリの場合、ファイルまたはアクティビティにガバナンス アクションを適用できます。 ガバナンスアクションは、ファイルまたはアクティビティに対して Microsoft Cloud App Security から直接実行できる統合アクションです。 ガバナンスアクションは、接続されているアプリ全体でユーザーがリアルタイムで実行する操作を制御します。

> [!NOTE]
> Microsoft Cloud App Security がファイルに対してガバナンスアクションを実行しようとしたときに、ファイルがロックされているために失敗した場合、ガバナンスアクションは自動的に再試行されます。

## <a name="file-governance-actions"></a>ファイル ガバナンス アクション

次のガバナンス アクションは、接続されたアプリで特定のファイル、ユーザーに対して、または特定のポリシーから実行できます。

- **警告**

  - **アラート**–警告は、システムでトリガーし、重大度レベルに基づいて電子メールとテキストメッセージを使用して伝達することができます。

  - **ユーザーの電子メール通知**–電子メールメッセージはカスタマイズ可能で、違反しているすべてのファイル所有者に送信されます。

  - **特定のユーザーに通知**する-これらの通知を受信する電子メールアドレスの特定のリスト。

  - **ファイルの最終編集**者に通知–ファイルを最後に変更したユーザーに通知を送信します。

- **アプリのガバナンスアクション**-詳細なアクションはアプリごとに適用でき、特定のアクションはアプリの用語によって異なります。

  - **ラベル付け**
    - Azure Information Protection 分類ラベルを追加するラベル機能を**適用**します。
    - **ラベルを削除**する-Azure Information Protection 分類ラベルを削除できます。
  - **共有の変更**

    - **パブリック共有の削除**–名前付きコラボレーターへのアクセスのみを許可します。たとえば、G Suite のパブリックアクセスを削除し、Box の直接共有リンクを削除します。

    - **外部ユーザーの削除**–会社のユーザーのみにアクセスを許可します。

    - [**プライベート**にする]: サイト管理者のみがファイルにアクセスでき、すべての共有が削除されます。

    - **コラボレーターの削除**–ファイルから特定のコラボレーターを削除します。

    - **パブリックアクセスの削減**-公開されているファイルを共有リンクでのみ使用できるように設定します。 同社

    - **共有リンクの有効期限が切れ**た場合は、共有リンクの experation 日付を設定して、アクティブにならないようにします。 箱

    - **共有リンクのアクセスレベルを変更**します。共有リンクのアクセスレベルを変更することができるのは、会社のみ、コラボレーターのみ、およびパブリックです。 箱

  - **検疫**

    - **ユーザー検疫に配置**–ユーザーによって制御される検疫フォルダーにファイルを移動してセルフサービスを許可します。

    - **管理者検疫に配置**–ファイルは管理ドライブで検疫に移動され、管理者はそれを承認する必要があります。

  - **親からの権限の継承**-このガバナンスアクションを使用すると、Office 365 でファイルまたはフォルダーに設定されている特定のアクセス許可を削除できます。 次に、親フォルダーに設定されているアクセス許可に戻します。

  - **ごみ箱**–ファイルを [ごみ箱] フォルダーに移動します。 (Box、Dropbox、Google Drive、OneDrive、SharePoint)

## <a name="activity-governance-actions"></a>アクティビティ ガバナンス アクション

- **警告**

  - **アラート**–警告は、システムでトリガーし、重大度レベルに基づいて電子メールとテキストメッセージを使用して伝達することができます。

  - **ユーザーの電子メール通知**–電子メールメッセージはカスタマイズ可能で、違反しているすべてのファイル所有者に送信されます。

  - **追加のユーザーに通知**する-これらの通知を受信する電子メールアドレスの特定のリスト。

- **アプリのガバナンスアクション**-詳細なアクションはアプリごとに適用でき、特定のアクションはアプリの用語によって異なります。

  - **[ユーザーの中断]** –アプリケーションからユーザーを中断します。
    > [!NOTE]
    > Azure Active Directory (Azure AD) が Active Directory オンプレミス環境のユーザーと自動的に同期するように設定されている場合、オンプレミス環境の設定は Azure AD の設定より優先され、このガバナンスアクションは元に戻されます。

  - **ユーザーに再度サインインを要求する**-ユーザーにサインアウトし、もう一度サインインする必要があります。

  - ユーザーが侵害された**ことを確認**する-ユーザーのリスクレベルを高に設定します。 これにより、Azure AD で定義された関連するポリシーアクションが適用されます。 Azure AD がリスクレベルで機能する方法の詳細については、「 [Azure AD でのリスクフィードバックの使用方法](https://docs.microsoft.com/azure/active-directory/identity-protection/howto-identity-protection-risk-feedback#how-does-azure-ad-use-my-risk-feedback)」を参照してください。

## <a name="governance-conflicts"></a>ガバナンスの競合

複数のポリシーを作成した後に、複数のポリシーに含まれるガバナンス アクションが重複する状況が発生することがあります。 その場合、Cloud App Security はガバナンス アクションを次のように処理します。

### <a name="conflicts-between-policies"></a>ポリシー間の競合

- あるポリシーのアクションに、別のポリシーのアクションが含まれている場合 (たとえば、**外部共有を削除する**は**非公開にする**に含まれます)、Cloud App Security は競合を解決して、強い方のアクションを適用します。
- アクションが関連付けられていない場合 (**所有者**への通知と**プライベートの作成**など)。 両方のアクションが実行されます。
- アクションが競合する場合 (たとえば、**所有者をユーザー a に変更**し、**所有者をユーザー B に変更**する場合)、一致ごとに結果が異なる可能性があります。 競合を防ぐために、ポリシーを変更することが重要です。これは、検出しにくいドライブに変更が生じる可能性があるためです。

### <a name="conflicts-in-user-sync"></a>ユーザー同期の競合

- Azure AD が Active Directory オンプレミス環境のユーザーと自動的に同期するように設定されている場合、オンプレミス環境の設定は Azure AD の設定よりも優先され、このガバナンスアクションは元に戻されます。

## <a name="governance-log"></a>ガバナンス ログ

ガバナンス ログは、Cloud App Security で実行するように設定された各タスク (手動と自動の両方) の状態レコードを提供します。 これらのタスクには、ポリシーで設定したものや、ファイルやユーザーに設定したガバナンスアクションや、実行する Cloud App Security 設定したその他のアクションなどがあります。 ガバナンス ログは、これらのアクションの成否に関する情報も提供します。 ガバナンス ログからは、ガバナンス アクションの一部を再試行したり、元に戻したりすることができます。
