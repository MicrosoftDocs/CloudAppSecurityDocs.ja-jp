# <a name="manage-app-permissions"></a>アプリのアクセス許可の管理
組織内のビジネス ユーザーがインストールするサード パーティ製の生産性アプリの多くでは、ユーザー情報とデータにアクセスし、Office 365 などの他のクラウド アプリにユーザーの代理としてサインインするためのアクセス許可が要求されます。  ユーザーは、こうしたアプリをインストールする場合、アプリへのアクセス許可の付与などの詳細を厳密に確認せずに、承認の操作を行いがちです。  IT 部門がこうしたアプリケーションのセキュリティ上のリスクと生産性のメリットとを比較検討するための十分な知識を持っていない場合、この問題はさらに重大になります。 サードパーティ製アプリのアクセス許可を承認することは、組織にとって潜在的なセキュリティ リスクになるため、ユーザーが付与するアプリのアクセス許可を監視することで、ユーザーとアプリケーションの保護に欠かせない可視性と制御が得られます。 Cloud App Security アプリのアクセス許可を使うと、ユーザーがインストールしたどのアプリケーションが Office 365 データに対するアクセスを持っているか、アプリがどんなアクセス許可を持っているか、どのユーザーがこれらアプリから Office 365 アカウントへのアクセスを付与したかがわかります。 アプリのアクセス許可は、ユーザーのアクセスを許可するアプリと、禁止するアプリを判断するのに役立ちます。


## <a name="working-with-the-app-permissions-page"></a>[アプリのアクセス許可] ページの操作

**[アプリのアクセス許可]** タブには、Office 365 テナントでのアプリのアクセス許可に関する情報が表示されます。

[アプリの権限] タブにアクセスするには、次の操作を実行します。

1. Cloud App Security ポータルで、**[調査]** をクリックし、[承認されたアプリ] で **Office 365** を選択します。 
> [!Note]
> [アプリのアクセス許可] を使用するには、まず Cloud App Security を Office 365 に接続する必要があります。

2. 次に、[Office 365] ダッシュボードで、**[アプリのアクセス許可]** タブをクリックします。


 ![アプリのアクセス許可](./media/app-permissions.png)

[アプリのアクセス許可] ページでは、アクセス許可が付与されているサードパーティ製アプリの各々について、下記の情報が表示されます。

|Item|意味|
|-------|----------------|
|アプリ クエリ バーの [基本] アイコン  |基本ビューでのクエリに切り替えるには、このアイコンをクリックします。|
|アプリ クエリ バーの [詳細] アイコン  |詳細ビューでのクエリに切り替えるには、このアイコンをクリックします。|
|アプリ一覧の [すべての詳細を開く/閉じる] アイコン  |各アプリに関する詳細の表示を切り替えるには、このアイコンをクリックします。|
|アプリ一覧の [エクスポート] アイコン  |このアイコンをクリックすると、アプリの一覧、各アプリのユーザー数、アプリに関連付けられているアクセス許可、アクセス許可のレベル、アプリの状態、コミュニティの利用状況レベルを含む CSV ファイルがエクスポートされます。|
|アプリ|アプリの名前です。 名前をクリックすると、説明、発行者、アプリの Web サイトおよび ID などを含む詳細が表示されます。|
|承認者|アプリの Office 365 アカウントへのアクセスを承認し、アプリのアクセス許可を付与したユーザーの数です。 数字をクリックすると、ユーザー電子メールの一覧、管理者がアプリに同意済みかどうかなどの詳細が表示されます。|
|アクセス許可のレベル  |アイコンとテキストでアクセス許可レベルが高、中、低のいずれかを示します。 このレベルは、このアプリケーションが Office 365 データに対して持っているアクセス権の数を示します。 たとえば、[低] の場合、アプリはユーザー プロファイルと名前にのみアクセスできます。 レベルをクリックすると、アプリに付与されているアクセス許可、コミュニティの利用状況、[ガバナンス ログ](governance-actions.md)の関連アクティビティなどの詳細が表示されます。|
|アプリの状態 - [禁止されています]、[承認済み]、[未定]  |管理者は、承認済みまた禁止としてマークするか、未確定のままにすることができます。|


## <a name="ban-or-approve-an-app"></a>アプリを禁止または承認する
1. [アプリのアクセス許可] ページで、アプリをクリックしてアプリ ドロワーを開き、アプリと付与されているアクセス許可に関する詳細情報を表示します。 [アクセス許可] リンクをクリックすると、そのアプリに付与されたアクセス許可の一覧が表示されます。 [コミュニティの利用状況]では、他の組織でそのアプリがどれだけ使われているかを確認できます。 [関連アクティビティ] リンクをクリックして、このアプリに関連するガバナンス ログに記載されているアクティビティを表示することもできます。
2. アプリのアクセスを禁止するには、テーブル内のアプリ行の末尾にある [禁止] アイコンをクリックします。 <br></br>
 ![[アプリの禁止] アイコン](./media/ban-app-icon.png) <br></br>
アプリのアクセスを禁止する場合、ユーザーに対して、以前インストールして承認したアプリが禁止および無効化され、Office 365 にアクセスできなくなることを通知するかどうかを選択できます。 通知しない場合は、[アプリの禁止] ダイアログにある [この禁止されたアプリにアクセス許可を付与したユーザーに通知を送信します。] をオフにします。

    ![アプリの禁止](./media/ban-app.png)
> [!Note]
> アプリのユーザーに、アプリが使用禁止になると通知することをお勧めします。

3. アプリを承認するには、テーブル内の行の末尾にある [承認] アイコンをクリックします。 <br></br>
 ![アプリの承認](./media/approve-app.png) <br></br>
アイコンが緑色に変わり、すべての Office 365 ユーザーに対してこのアプリが承認されます。
> [!Note]
> アプリを承認済みとしてマークしても、エンドユーザーへの影響はありません。 これは承認済みのアプリを視覚的にマークして、まだ確認していないものと区別するためのものです。

3. [カスタム通知メッセージの入力] ボックスで、アプリのユーザーに送信するメッセージを入力し、通知電子メールの "返信先" アドレスを必要に応じて更新します。 
 **[アプリの禁止]** をクリックすると、メールが送信され、Office 365 ユーザーのアプリ使用が禁止されます。


## <a name="query-app-permissions"></a>アプリのアクセス許可のクエリ

### <a name="query-in-the-advanced-view"></a>詳細ビューでのクエリ 
1. 詳細ビューでは、**[フィルターの選択]** ドロップダウンを使って検索を絞り込みむことができます。 選択した値に対して [が次と等しい] 演算子または [が次に等しくない] 演算子を追加し、クエリを完成させます。
2. クエリの精度を上げるには、**[フィルターを追加]** アイコンをクリックしてフィルターをさらに追加します。 フィルターは自動的に適用され、アプリの一覧がそれに応じて更新されます。
3. フィルターを削除するには、そのフィルターの横にある **フィルターの削除**アイコンをクリックします。
選択できるフィルターは以下の通りです。
- アプリ: Office 365 へのアクセスが許可されている、選択した名前に該当する 1 つまたは複数のサードパーティ アプリを表示します。

- ユーザー: 指定したユーザーが承認したアプリを表示します。

- アクセス許可: 選択したアクセス許可が必要なアプリを表示します。

- アプリの状態: アプリの状態 ([承認済み]、[禁止されています]、[未定]) に基づいてアプリを表示します。

- アクセス許可レベル: 選択した 1 つまたは複数のアクセス許可レベルに基づいてアプリを表示します。

- コミュニティの利用状況: [珍しい]、[一般的でない]、[共通] のいずれかのコミュニティの利用状況レベルに基づいてアプリを表示します。

### <a name="query-in-the-basic-view"></a>基本ビューでのクエリ 
基本ビューでは、1 つまたは複数のドロップダウン リストから値を選択して、特定のアプリを表示できます。 ドロップダウン リストでは、複数の値を選択できます。 クエリに使用することができるドロップダウン メニューは次のとおりです。 
- アプリ: Office 365 へのアクセスが許可されている、選択した名前に該当する 1 つまたは複数のサードパーティ アプリを表示します。

- ユーザー: 指定したユーザーが承認したアプリを表示します。

- アクセス許可: 選択したアクセス許可が必要なアプリを表示します。

- アプリの状態: アプリの状態 ([承認済み]、[禁止されています]、[未定]) に基づいてアプリを表示します。

- アクセス許可レベル: 選択した 1 つまたは複数のアクセス許可レベルに基づいてアプリを表示します。

- コミュニティの利用状況: [珍しい]、[一般的でない]、[共通] のいずれかのコミュニティの利用状況レベルに基づいてアプリを表示します。
フィルターは自動的に適用され、アプリの一覧がそれに応じて更新されます。
