# <a name="manage-oauth-apps"></a>OAuth アプリの管理

*適用対象: Microsoft Cloud App Security*

組織内のビジネスユーザーによってインストールされる可能性のあるサードパーティ製生産性向上アプリの多くは、ユーザー情報とデータにアクセスし、Office 365、G Suite、Salesforce などの他のクラウドアプリのユーザーに代わってサインインするアクセス許可を要求します。 ユーザーは、これらのアプリをインストールするときに、アプリへのアクセス許可の付与など、プロンプトの詳細を厳密に確認せずに [同意する] をクリックすることがよくあります。 この問題は、アプリケーションのセキュリティリスクを、提供されている生産性向上と比較するのに十分な洞察を得られない可能性があるため、複雑になります。 サードパーティのアプリのアクセス許可を受け入れることは組織にとって潜在的なセキュリティリスクであるため、ユーザーに付与されているアプリのアクセス許可を監視することで、ユーザーとアプリケーションを保護するために必要な可視性と制御が得られます。 Microsoft Cloud App Security アプリのアクセス許可を使用すると、ユーザーがインストールした OAuth アプリケーションで、Office 365 データ、G Suite データ、および Salesforce データにアクセスできるかどうかを確認できます。 Cloud App Security は、アプリのアクセス許可と、それらのアプリが Office 365、G Suite、Salesforce アカウントにアクセスすることを許可したユーザーを示します。 アプリのアクセス許可は、ユーザーがアクセスを許可するアプリと禁止するアプリを決定するのに役立ちます。

OAuth アプリの調査の詳細については、「リスクのある[oauth アプリの調査](investigate-risky-oauth.md)」を参照してください。

[OAuth] タブにアクセスするには:

![アプリのアクセス許可](media/app-permissions.png)

## <a name="ban-or-approve-an-app"></a>アプリの禁止または承認

1. **[OAuth アプリ]** ページで、アプリをクリックしてアプリ**ドロワー**を開き、アプリとそれに付与されたアクセス許可に関する詳細情報を表示します。

    - **[アクセス許可]** リンクをクリックすると、アプリに付与されたアクセス許可の完全な一覧が表示されます。
    - [**コミュニティ**での使用] では、他の組織におけるアプリの共通点を確認できます。
    - **[関連アクティビティ]** リンクをクリックすると、このアプリに関連するガバナンスログに一覧表示されているアクティビティが表示されます。

2. アプリの禁止を解除するには、テーブルのアプリ行の末尾にある [禁止] アイコンをクリックします。

    ![アプリの禁止アイコン](media/ban-app-icon.png)

    - インストールされているアプリと承認されたアプリをユーザーに通知するかどうかを選択できます。 通知では、アプリが無効になり、接続されているアプリにアクセスできなくなることをユーザーに通知します。 ユーザーに知らせたくない場合は、**この禁止されたアプリへのアクセスを許可**したユーザーにダイアログで通知することを選択解除します。
    - アプリのユーザーにアプリの使用を禁止することを知らせることをお勧めします。

    ![アプリの禁止](media/ban-app.png)

3. [カスタム通知メッセージを入力してください] ボックスに、アプリユーザーに送信するメッセージを入力します。 **[アプリの禁止]** をクリックしてメールを送信し、接続されているアプリユーザーからアプリの使用を禁止します。

4. アプリを承認するには、テーブルの行の末尾にある [承認] アイコンをクリックします。

    ![アプリの承認](media/approve-app.png)

    - アイコンが緑色に変わり、アプリは接続されているすべてのアプリユーザーに対して承認されます。
    - アプリを承認済みとしてマークすると、エンドユーザーに影響はありません。 この色の変更は、まだレビューしていないアプリと区別するために承認したアプリを表示するのに役立ちます。

## <a name="revoke-app-and-notify-user"></a>アプリの取り消しとユーザーへの通知

G Suite と Salesforce では、アプリへのアクセス許可を取り消したり、アクセス許可を変更する必要があることをユーザーに通知したりすることができます。 アクセス許可を取り消すと、Azure AD の [エンタープライズアプリケーション] で、アプリケーションに付与されたすべてのアクセス許可が削除されます。

1. **[OAuth アプリ]** ページで、アプリの行の末尾にある3つの点をクリックし、ユーザーへの **[通知]** を選択します。 既定では、ユーザーには次の通知が表示されます。 *G Suite アカウントへのアクセスをアプリに許可した。このアプリは、組織のセキュリティポリシーと競合しています。G Suite アカウントにこのアプリを付与したアクセス許可を付与するか取り消すことを再検討します。アプリのアクセスを取り消すには、: https://security.google.com/settings/security/permissions?hl=en&pli=1 アプリを選択し、右側のメニューバーの [アクセスの取り消し] をクリックします。* 送信されるメッセージをカスタマイズできます。
2. また、ユーザーにアプリを使用するためのアクセス許可を取り消すこともできます。 テーブルのアプリ行の末尾にあるアイコンをクリックし、 **[アプリの取り消し]** を選択します。

    ![アプリの取り消し](media/revoke-app.png)

## <a name="query-oauth-apps"></a>OAuth アプリのクエリ

OAuth アプリは、**基本**ビューまたは**詳細**ビューのいずれかで照会できます。 1つまたは複数のドロップダウンリストから値を選択して、基本ビューに特定のアプリを表示します。 [詳細ビュー] で、 **[フィルターの選択**] ドロップダウンを使用して検索を絞り込みます。 クエリを完了するために、選択した値に演算子、equals、または不等号を追加します。

- フィルターを追加するには、 **[フィルターの追加]** アイコンをクリックして、クエリをさらに絞り込むためのフィルターを追加します。 フィルターが自動的に適用され、アプリの一覧が更新されます。

- フィルターを削除するには、フィルターの横にある **[フィルターの削除]** アイコンをクリックします。

## <a name="oauth-app-auditing"></a>OAuth アプリの監査

Cloud App Security では、実行されたアクティビティの包括的な監視および調査を提供できるよう、すべての OAuth 承認アクティビティを監査しています。 また、特定の OAuth アプリを承認したユーザーの詳細をエクスポートして、ユーザーに関する追加情報を提供することもできます。これを使用すると、さらに詳しい分析を行うことができます。

ログをエクスポートするには、次の手順を実行します。

1. **[OAuth アプリ]** ページで、関連するアプリが表示されている行の **[承認者]** の下で、アプリを承認したユーザーの数を示すリンクをクリックします。

1. ポップアップウィンドウで、 **[エクスポート]** をクリックします。

    ![OAuth アプリの監査のエクスポートを示すスクリーンショット](media/oauth-export-users.png)

## <a name="send-feedback"></a>フィードバックの送信

組織で検出された OAuth アプリが悪意があると思われる場合は、Cloud App Security チームのフィードバックを送信してお知らせください。 この機能を使用すると、セキュリティコミュニティに参加し、OAuth アプリのリスクスコアと分析を強化することができます。

1. **[OAuth アプリ]** ページで、アプリの行の末尾にある3つの点をクリックし、 **[レポートアプリ]** を選択します。

    ![レポートアプリ](media/report-app.png)
2. [**このアプリをレポート**する] 画面で、アプリを悪意のあるものとして報告するか、またはアプリを認識 Cloud App Security 方法で別の問題を報告するかを選択できます。 たとえば、**不適切な発行者**、不適切な**権限**、または**その他の**を使用できます。 送信したデータは、アプリのリスクスコアとアプリに関するその他の分析を更新するために使用されます。
