# <a name="manage-app-permissions"></a>アプリのアクセス許可の管理
組織内のビジネス ユーザーがインストールするサード パーティ製の生産性アプリの多くでは、ユーザー情報とデータにアクセスし、Office 365、G Suite、Salesforce などの他のクラウド アプリにユーザーの代理としてサインインするためのアクセス許可が要求されます。  ユーザーは、こうしたアプリをインストールする場合、アプリへのアクセス許可の付与などの詳細を厳密に確認せずに、承認の操作を行いがちです。  IT 部門がこうしたアプリケーションのセキュリティ上のリスクと生産性のメリットとを比較検討するための十分な知識を持っていない場合、この問題はさらに重大になります。 サード パーティ製アプリのアクセス許可を承認することは、組織にとって潜在的なセキュリティ リスクになるため、ユーザーが付与するアプリのアクセス許可を監視することで、ユーザーとアプリケーションの保護に欠かせない可視性と制御が得られます。 Microsoft Cloud App Security アプリのアクセス許可を使用すると、ユーザーがインストールしたどのアプリケーションが Office 365 データ、G Suite データおよび Salesforce データに対するアクセスを持っているか、アプリがどんなアクセス許可を持っているか、どのユーザーがこれらのアプリから Office 365、G Suite および Salesforce アカウントへのアクセスを付与したかがわかります。 アプリのアクセス許可は、ユーザーのアクセスを許可するアプリと、禁止するアプリを判断するのに役立ちます。


## <a name="working-with-the-app-permissions-page"></a>[アプリのアクセス許可] ページの操作

**[アプリのアクセス許可]** ページには、接続されたアプリでのアプリのアクセス許可に関する情報が表示されます。

[アプリの権限] タブにアクセスするには、次の操作を実行します。

Cloud App Security ポータルで、**[調査]**、**[アプリのアクセス許可]** の順にクリックします。


 ![アプリのアクセス許可](./media/app-permissions.png)

[アプリのアクセス許可] ページでは、アクセス許可が付与されているサードパーティ製アプリの各々について、下記の情報が表示されます。

|Item|意味|適用対象|
|-------|-------|-------|
|アプリ クエリ バーの [基本] アイコン  |基本ビューでのクエリに切り替えるには、このアイコンをクリックします。|Office 365、G Suite、Salesforce|
|アプリ クエリ バーの [詳細] アイコン  |詳細ビューでのクエリに切り替えるには、このアイコンをクリックします。|Office 365、G Suite、Salesforce|
|アプリ一覧の [すべての詳細を開く/閉じる] アイコン  |各アプリに関する詳細の表示を切り替えるには、このアイコンをクリックします。|
|アプリ一覧の [エクスポート] アイコン  |このアイコンをクリックすると、アプリの一覧、各アプリのユーザー数、アプリに関連付けられているアクセス許可、アクセス許可のレベル、アプリの状態、コミュニティの利用状況レベルを含む CSV ファイルがエクスポートされます。|Office 365、G Suite、Salesforce|
|アプリ|アプリの名前です。 名前をクリックすると、説明、発行者 (Office 365 の場合)、アプリの Web サイト、ID などを含む詳細が表示されます。|Office 365、G Suite、Salesforce|
|承認者|アプリのアカウントへのアクセスを承認し、アプリのアクセス許可を付与したユーザーの数です。 数字をクリックすると、ユーザー電子メールの一覧、管理者がアプリに同意済みかどうかなどの詳細が表示されます。|Office 365、G Suite、Salesforce|
|アクセス許可のレベル  |アイコンとテキストでアクセス許可レベルが高、中、低のいずれかを示します。 このレベルは、このアプリケーションがアプリのデータに対して持っているアクセス権の数を示します。 たとえば、[低] の場合、アプリはユーザー プロファイルと名前にのみアクセスできます。 レベルを選択すると、アプリに付与されているアクセス許可、コミュニティの利用状況、[ガバナンス ログ](governance-actions.md)の関連アクティビティなどの詳細が表示されます。|Office 365、G Suite|
|アプリの状態|管理者は、承認済みまた禁止としてマークするか、未確定のままにすることができます。|Office 365、G Suite、Salesforce|
|コミュニティの利用状況|すべてのユーザーの間でアプリがどれくらい使われいてるか (一般的に使われている、一般的に使われていない、ほとんど使われていない) を把握したい場合は、これを選択します。|Office 365、G Suite、Salesforce|
|最終承認日|ユーザーがこのアプリにアクセス許可を付与した最新の日付。|Office 365、Salesforce|
|発行元|アプリを提供するベンダーの名前。|Office 365|
|最終使用日|組織内のだれかがこのアプリを使用した最新の日付。|Salesforce|


## <a name="ban-or-approve-an-app"></a>アプリを禁止または承認する
1. [アプリのアクセス許可] ページで、アプリをクリックしてアプリ ドロワーを開き、アプリと付与されているアクセス許可に関する詳細情報を表示します。 [アクセス許可] リンクをクリックすると、そのアプリに付与されたアクセス許可の一覧が表示されます。 [コミュニティの利用状況] では、他の組織でそのアプリがどれだけ使われているかを確認できます。 [関連アクティビティ] リンクをクリックして、このアプリに関連するガバナンス ログに記載されているアクティビティを表示することもできます。
2. アプリのアクセスを禁止するには、テーブル内のアプリ行の末尾にある [禁止] アイコンをクリックします。 <br></br>
   ![[アプリの禁止] アイコン](./media/ban-app-icon.png) <br></br>
   アプリのアクセスを禁止する場合、ユーザーに対して、以前インストールして承認したアプリが禁止および無効化され、接続されたアプリにアクセスできなくなることを通知するかどうかを選択できます。 通知しない場合は、[アプリの禁止] ダイアログにある [この禁止されたアプリにアクセス許可を付与したユーザーに通知を送信します。] をオフにします。

    ![アプリの禁止](./media/ban-app.png)
   > [!Note]
   > アプリのユーザーに、アプリが使用禁止になると通知することをお勧めします。

3. アプリを承認するには、テーブル内の行の末尾にある [承認] アイコンをクリックします。 <br></br>
   ![アプリの承認](./media/approve-app.png) <br></br>
   アイコンが緑色に変わり、接続されたすべてのアプリ ユーザーに対してこのアプリが承認されます。
   > [!Note]
   > アプリを承認済みとしてマークしても、エンドユーザーへの影響はありません。 これは承認済みのアプリを視覚的にマークして、まだ確認していないものと区別するためのものです。

4. [カスタム通知メッセージの入力] ボックスで、アプリのユーザーに送信するメッセージを入力し、通知電子メールの "返信先" アドレスを必要に応じて更新します。 
   **[アプリの禁止]** をクリックすると、メールが送信され、接続されたアプリ ユーザーのアプリ使用が禁止されます。

## <a name="revoke-app-and-notify-user"></a>アプリを取り消し、ユーザーに通知する

G Suite と Salesforce の場合、アプリのアクセス許可を取り消したり、禁止行為であることをユーザーに通知したりできます。 

1. アプリのアクセス許可ページで、アプリの行の終わりにある 3 つの点をクリックし、**[ユーザーに通知]** を選択します。 既定では、ユーザーに次の通知が届きます。*You authorized the app Adallom Google Protector to access your G Suite account.このアプリはあなたが所属する組織のセキュリティ ポリシーに相反します。自分の G Suite アカウントにアクセスする許可を与えるべきか、取り消すべきか再検討してください。アプリのアクセスを取り消すには、https://security.google.com/settings/security/permissions?hl=en&pli=1 に移動して、アプリを選択し、右側のメニュ ーバーの Revoke access をクリックします。* 送信メッセージはカスタマイズできます。
2. ユーザーのためにアプリを使用するアクセス許可を取り消すこともできます。テーブルのアプリの行の終わりにあるアイコンをクリックし、**[アプリの取り消し]** を選択します。 

   ![アプリを取り消す](./media/revoke-app.png)

## <a name="query-app-permissions"></a>アプリのアクセス許可のクエリ

**[基本]** ビューまたは **[詳細]** ビューで、アプリのアクセス許可のクエリを実行することができます。 [基本] ビューでは、1 つまたは複数のドロップダウン リストから値を選択して、特定のアプリを表示できます。 詳細ビューでは、**[フィルターの選択]** ドロップダウン リストを使って検索を絞り込むことができます。 選択した値に対して [が次と等しい] 演算子または [が次に等しくない] 演算子を追加し、クエリを完成させます。

- クエリの精度を上げるには、**[フィルターを追加]** アイコンをクリックしてフィルターをさらに追加します。 フィルターは自動的に適用され、アプリの一覧がそれに応じて更新されます。

- フィルターを削除するには、そのフィルターの横にある **フィルターの削除**アイコンをクリックします。
