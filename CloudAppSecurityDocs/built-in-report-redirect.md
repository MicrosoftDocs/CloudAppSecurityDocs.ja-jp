# <a name="how-to-find-built-in-deprecating-reports"></a>非推奨になる組み込みレポートを検索する方法

*適用対象:Microsoft Cloud App Security*

Microsoft では、組み込みレポートの機能をポータルの他の部分に埋め込むことによって更新しています。 この機能の更新は、Microsoft Cloud App Security レポートを改善するために継続されています。

## <a name="deprecating-reports"></a>非推奨になるレポート

次の表は、非推奨になるレポートによって提供されていた情報を、他の Cloud App Security 機能を使用して表示する場合に役立ちます。

| レポートの種類 | 組み込みレポートの名前 | [説明] | データの新しい場所 |
|----|----|----|----|
| セキュリティ | アクティビティ | このレポートでは、クラウド アプリ内で発生したアクティビティの発生元の国の一覧を表示できます。 レポートには、イベントの数やユーザーの数など、各国のアクティビティの量を明らかにするさまざまなパラメーターが表示されます。 ユーザーの地理的な分散を大まかに把握するのに役立ちます。 | この情報は、特定のアプリ、ユーザー、IP アドレスについて入手できます。 各アプリについては、アプリをクリックします。 そこで、すべての場所を含むアクティビティ マップが表示されます。 各ユーザーについては、ユーザーをクリックし、ユーザー分析情報ドロワーをクリックします。 最近使用された場所と、使用された IP アドレスと ISP の数が表示されます。 各 IP アドレスについては、特定の IP アドレスをクリックします。 IP アドレス ドロワーに、その IP アドレスを使用したユーザーと管理者の数が表示されます。 |
| セキュリティ | ブラウザーの使用 | ブラウザーベースの攻撃は、最も一般的な攻撃ベクトルの 1 つです。 ベンダーは、参照ソフトウェアのセキュリティ保護に膨大な量のリソースを投資し、更新をエンドポイントに発行するための効果的な更新メカニズムを作成しています。 更新されてから長期間経過した非推奨のブラウザーを使用すると、悪意のある人物が入手可能な攻撃キットを使用して容易に攻撃できるようになります。 この機能により、過去 7 日間にユーザーがクラウド アプリにアクセスするときに使用した古いブラウザーの一覧を取得できます。 また、古いブラウザーの使用がロボットによって実行されたかどうかも確認できます。 | **[アクティビティ ログ]** に移動し、 **[高度なフィルター]** を開きます。 **[ユーザー エージェント タグ]** が **[古いブラウザー]** と **[古いオペレーティング システム]** と等しくなるようにフィルターを設定して、使用されているすべての古いブラウザーおよび OS の一覧を表示します。 |
| セキュリティ | IP アドレス - 特権アカウント| このレポートには、Cloud App Security で保護されているクラウド環境で過去 7 日間に管理アクティビティを行ったデバイスで使用された IP アドレスの一覧が表示されます。 日付は、Cloud App Security により蓄積された監査ログに基づいています。 IP アドレスは、地理的な場所および送信元の組織に関連付けられています。 この情報を使用すると、保護されたアプリへの接続に使用された不審な IP アドレスを識別できます。 IP アドレスをクリックすると、各 IP アドレスの監査ログを表示できます。 | 特定のユーザーをクリックします。 ユーザー分析情報ドロワーに、最近使用された場所と、使用された IP アドレスと ISP の数が表示されます。 |
| ユーザー管理 | 非アクティブなアカウント | 非アクティブなアカウントとは、クラウド インスタンスにアクセスすることができ、しかも過去 60 日間何のイベントも実行していないアカウントのことです。 アクションがないということは、それらのアカウントがアクティブではなくなっていることを意味するため、今後悪意のある人物や退職した従業員がアクセスできないようにするために、一時停止する必要があります。 このベスト プラクティスに従うことにより、セキュリティ体制を強化するだけでなく、運用コストを削減することができます。 | **[ユーザーとアカウント]** ページに移動し、 **[最終接続]** フィルターを使用して、最近アクティビティをまったく実行していないすべてのユーザーの一覧を生成します。 |
| ユーザー管理 | 特権のあるユーザー | このレポートには、過去 7 日間に会社のアプリで権限が昇格されたユーザー (たとえば、管理者など) のリストが表示されます。 このような特権アカウントは、企業の情報やネットワーク構成に対して高い権限でアクセスできるため、悪意のある人物の攻撃ベクトルとして使用される可能性があります。 最近使用されていない特権アカウントがある場合は、IT セキュリティに対する企業の認識が不十分であることを示しており、データ侵害が次々に発生する危険性があります。 監査ログを使用して、昇格されたユーザー特権の使用状況を詳細に調査し、不要な場合は権限を取り消すことを検討してください。 | **[ユーザーとアカウント]** ページに移動し、 **[グループ]** フィルターを使用して、Administrators グループに属する、特権のあるユーザーの一覧を生成します。 |
| ユーザー管理 | 特別な特権アカウント | Salesforce には、すべてのデータの変更、すべてのデータの表示、すべてのユーザーの管理など、複数の種類の特権アカウントがあります。 このような特権アカウントは、企業の情報やネットワーク構成に対して高い権限でアクセスできるため、悪意のある人物の攻撃ベクトルとして使用される可能性があります。このため、特権アカウントの一覧を表示することが有用です。 | [Salesforce] に移動します。 **[特別な特権アカウント]** タブをクリックします。 |
| データ管理 | データ共有の概要 | このレポートには、クラウド アプリに保存されているファイルの数が、アクセス許可ごとに分けられて表示されます。 クラウド アプリでは、アクセスが容易でどこからでも利用できるため、共有が簡単にできるようになりました。 ファイルの所有者以外のユーザーと共有されないファイルは、プライベート ファイルと呼ばれます。 ファイルが共有されている場合、Cloud App Security は 4 種類の状態を区別します。 <br /> - 公開共有 (Web) ファイルは、認証を受けなくても検索エンジンの結果などからアクセスできるファイルです。<br /> - 公開共有ファイルは、認証を受けなくてもリンクを使用してアクセスできるファイルです。<br /> - 外部共有ファイルは、組織外のユーザーがクラウド アプリの認証を受けてアクセスできるファイルです。<br /> - 内部共有ファイルは、組織内のすべてまたは一部のユーザーがアクセスできるファイルです。|**[ファイル]** に移動します。 右上隅にある 3 つのドットをクリックし、 **[データ管理レポート]** で、 **[データ共有の概要]** を選択します。 |
| データ管理 | ドメイン別の送信共有 | このレポートでは、従業員が共有している企業ファイルがあるドメインが一覧表示されます。 レポートには、ドメインごとに、そのドメインでファイルを共有している企業ユーザー、共有されているファイル、ファイルを共有しているコラボレーターの詳細が表示されます。 関連する各アプリの [アプリ] ページの [ファイル] タブで、これらのドメインでの共有を管理することをお勧めします。 | **[ファイル]** に移動します。 右上隅にある 3 つのドットをクリックし、 **[データ管理レポート]** で、 **[ドメイン別の送信共有]** を選択します。 |
| データ管理 | 共有ファイルの所有者 | これには、企業ファイルを外部と共有しているユーザーが一覧表示されます。 外部共有ファイルは、特定の外部のコラボレーターと共有されます。 公開共有ファイルには、プライベート リンクが明示的に公開されているユーザーであれば誰でも、そのリンクを使用して、インターネット上でアクセスすることができます。 公開共有ファイル (インターネット) は、検索エンジンの結果を使用してインターネット上で誰でもアクセスすることができます。 過剰な数のファイルを共有しているユーザーが見つかった場合、[ファイル] タブを使用して、それらの過剰な共有アクセス許可の性質を調査し、ユーザーに連絡して、外部共有の使用状況を詳細に把握することをお勧めします。 | **[ファイル]** に移動します。 右上隅にある 3 つのドットをクリックし、 **[データ管理レポート]** で、 **[共有ファイルの所有者]** を選択します。 |
