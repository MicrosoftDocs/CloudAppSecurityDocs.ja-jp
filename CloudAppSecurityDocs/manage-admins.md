# <a name="manage-admin-access"></a>管理者アクセスの管理

*適用対象: Microsoft Cloud App Security*

Microsoft Cloud App Security はロールベースのアクセス制御に対応しています。 この記事では、管理者の Cloud App Security ポータルへのアクセスを設定する手順について説明します。 管理者ロールの割り当ての詳細については、 [Azure Active Directory](https://docs.microsoft.com/azure/active-directory/active-directory-assign-admin-roles)と[Office 365](https://docs.microsoft.com/office365/admin/add-users/assign-admin-roles)の記事を参照してください。

## <a name="office-365-and-azure-ad-roles-with-access-to-cloud-app-security"></a>Cloud App Security にアクセスできる Office 365 および Azure AD ロール

既定では、次の Office 365 および[Azure Active Directory (Azure AD)](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles)の管理者ロールは Cloud App Security にアクセスできます。

- **全体管理者とセキュリティ管理者:** **フルアクセス**権を持つ管理者は、Cloud App Security の完全な権限を持ちます。 管理者は、管理者の追加、ポリシーと設定の追加、ログのアップロード、ガバナンスアクションの実行を行うことができます。

- **コンプライアンス管理者:** 読み取り専用アクセス許可を持ち、アラートを管理できます。 ファイルポリシーを作成および変更したり、ファイルガバナンスアクションを許可したり、データ管理の下にあるすべての組み込みレポートを表示したりできます。

- **コンプライアンスデータ管理者:** 読み取り専用アクセス許可を持ち、ファイルポリシーの作成と変更、ファイルガバナンスアクションの許可、およびすべての検出レポートの表示を行うことができます。

- **セキュリティオペレーター:** 読み取り専用アクセス許可を持ち、アラートを管理できます。

- **セキュリティ閲覧者:** 読み取り専用アクセス許可を持ち、アラートを管理できます。 セキュリティ閲覧者は、次の操作の実行を制限されています。

  - ポリシーを作成する、または既存のポリシーを編集および変更する
  - ガバナンス アクションを実行する
  - 探索ログをアップロードする
  - サードパーティ製アプリの禁止または承認
  - IP アドレス範囲設定ページへのアクセスと表示
  - すべての設定ページへのアクセスと表示
  - 探索設定へのアクセスと表示
  - アプリコネクタページへのアクセスと表示
  - ガバナンス ログへのアクセスと表示
  - [スナップショット レポートの管理] ページへのアクセスと表示
  - SIEM エージェントへのアクセスと編集

- **グローバルリーダー:** Microsoft Cloud App Security のすべての側面に対する完全な読み取り専用アクセス権を持ちます。 設定を変更したり、アクションを実行したりすることはできません。

また、Cloud App Security ポータルでは、次の Cloud App Security 特定の管理者ロールを構成できます。

- **アプリ/インスタンス管理者:** には、選択した特定のアプリまたはインスタンスのみを扱う Microsoft Cloud App Security のすべてのデータに対するフルアクセスまたは読み取り専用のアクセス許可があります。 たとえば、Box の欧州インスタンスに対する管理者権限をユーザーに付与したとします。 管理者は、ファイル、アクティビティ、ポリシー、アラートのいずれであるかにかかわらず、欧州の箱のインスタンスに関連するデータのみを表示します。

  - アクティビティページ-特定のアプリに関するアクティビティ
  - アラート-特定のアプリに関連するアラートのみ
  - ポリシー-すべてのポリシーを表示できます。割り当てられた完全なアクセス許可では、アプリ/インスタンスを排他的に扱うポリシーのみを編集または作成できます。
  - [アカウント] ページ-特定のアプリ/インスタンスのアカウント
  - アプリのアクセス許可-特定のアプリ/インスタンスに対するアクセス許可のみ
  - ファイルページ-特定のアプリ/インスタンスのファイルのみ
  - アプリの条件付きアクセス制御-アクセス許可なし
  - Cloud Discovery アクティビティ-アクセス許可なし
  - セキュリティ拡張機能-ユーザーアクセス許可を持つ API トークンのアクセス許可のみ
  - ガバナンスアクション-特定のアプリ/インスタンスに対してのみ

- **ユーザーグループ管理:** ここで選択した特定のグループとのみ関連する Microsoft Cloud App Security のすべてのデータに対するフルアクセスまたは読み取り専用権限を持っています。 たとえば、"ドイツ-all users" というグループに "管理者" アクセス許可を付与した場合、管理者はそのユーザーグループに対してのみ Microsoft Cloud App Security の情報を表示および変更できます。

  - [活動] ページ-グループ内のユーザーに関するアクティビティ
  - アラート-グループ内のユーザーに関連するアラートのみ
  - ポリシー-すべてのポリシーを表示できます。割り当てられた完全なアクセス許可では、グループ内のユーザーのみを処理するポリシーのみを編集または作成できます。
  - [アカウント] ページ-グループ内の特定のユーザーのアカウント
  - アプリのアクセス許可–アクセス許可なし
  - [ファイル] ページ-アクセス許可なし
  - アプリの条件付きアクセス制御-アクセス許可なし
  - Cloud Discovery アクティビティ-アクセス許可なし
  - セキュリティ拡張機能-グループ内のユーザーと API トークンのアクセス許可のみ
  - ガバナンスアクション-グループ内の特定のユーザーに対してのみ

- **Cloud Discovery グローバル管理者:** すべての Cloud Discovery 設定とデータを表示および編集する権限があります。 グローバル検出管理者は、次のようにアクセスできます。

  - 設定
    - システム設定-表示のみ
    - Cloud Discovery の設定-[すべて表示] と [すべて編集] (匿名化権限は、ロールの割り当て中に許可されたかどうかによって異なります)
  - Cloud Discovery アクティビティ-完全なアクセス許可
  - アラート-Cloud Discovery データに関連するアラートのみ
  - ポリシー-すべてのポリシーを表示でき、Cloud Discovery ポリシーのみを編集または作成できます。
  - [活動] ページ-アクセス許可なし
  - [アカウント] ページ-アクセス許可なし
  - アプリのアクセス許可–アクセス許可なし
  - [ファイル] ページ-アクセス許可なし
  - アプリの条件付きアクセス制御-アクセス許可なし
  - セキュリティ拡張機能-アクセス許可なし
  - ガバナンスアクション-Cloud Discovery 関連のアクションのみ

- **レポート管理者の Cloud Discovery:** には、選択した特定の Cloud Discovery レポートだけを扱う Microsoft Cloud App Security のすべてのデータを表示する権限があります。 たとえば、Microsoft Defender ATP からの継続的なレポートに対する管理者権限を他のユーザーに与えることができます。 探索管理者には、そのデータソースとアプリカタログに関連する Cloud Discovery データのみが表示されます。
この管理者は、**アクティビティ**または**ファイル**のページにアクセスしたり、ポリシーへのアクセスを制限したりすることはできません。

## <a name="override-admin-permissions"></a>管理アクセス許可を上書きする

Azure Active Directory または Office 365 から管理者のアクセス許可を上書きする場合は、ユーザーを手動で Cloud App Security に追加し、ユーザーのアクセス許可を割り当てることによって、この操作を行うことができます。
たとえば、Cloud App Security で**フルアクセス**を行う Azure Active Directory のセキュリティ閲覧者である Stephanie を割り当てる場合は、ユーザーを手動で Cloud App Security に追加し、自分のロールを上書きして、Cloud App Security で必要なアクセス許可を付与するために自分の**フルアクセス**を割り当てることができます。

## <a name="add-additional-admins"></a>追加の管理者の追加

Azure Active Directory 管理者ロールにユーザーを追加せずに、Cloud App Security に管理者を追加できます。 さらに管理者を追加するには、次の手順を実行します。

> [!IMPORTANT]
> グローバル管理者またはセキュリティ管理者だけが、他のユーザーに Cloud App Security のアクセス権を付与できます。

1. [設定] 歯車![設定アイコン](media/settings-icon.png "設定アイコン")をクリックし、[管理**者アクセスの管理**] をクリックします。

2. プラス記号をクリックして、Cloud App Security へのアクセス権を持つ管理者を追加します。 内部または外部の電子メールアドレスを入力して、管理者が組織内または外部の管理対象セキュリティサービスプロバイダー (MSSPs) を使用してセキュリティ警告を管理できるようにすることができます。

    ![管理者の追加](media/add-admin.png)

3. 次に、ドロップダウンリストをクリックして、管理者が持つロールの種類、**グローバル管理者**、**セキュリティ閲覧**者、**コンプライアンス管理者**、または**アプリ/インスタンス管理者**を設定します。 **[アプリ/インスタンス管理者]** を選択した場合は、管理者がアクセス許可を持つアプリとインスタンスを選択します。

    >[!NOTE]
    >制限付きのページへのアクセスまたは制限されたアクションの実行を試みる管理者は、ページへのアクセスまたはアクションの実行を許可されていないというエラーを受け取ります。

4. **[管理者の追加]** をクリックします。

## <a name="admin-activity-auditing"></a>管理アクティビティの監査

Cloud App Security を使用すると、特定のユーザーを調査したり特定のアラートを表示したりする管理者の監査を含む、すべての管理者アクティビティのログをエクスポートできます。

ログをエクスポートするには、次の手順を実行します。

1. **[管理者アクセスの管理]** ページで、 **[管理者アクティビティのエクスポート]** を選択します。

1. 必要な時間の範囲を指定します。

1. **[エクスポート]** をクリックします。

## <a name="invite-external-admins"></a>外部管理者の招待

Cloud App Security を使用すると、Cloud App Security ポータルの管理者として、外部の管理対象セキュリティサービスプロバイダー (MSSPs) を招待することができます。 外部ユーザーを管理者として構成し、Cloud App Security で使用可能な任意のロールを割り当てることができるようになりました。 さらに、MSSPs で複数の顧客テナントにサービスを提供できるようにするために、複数のテナントへのアクセス権を持つ管理者は、ポータル内でテナントを簡単に切り替えることができるようになりました。

テナントを切り替えるには、複数のテナントに対するアクセス許可を取得した後、[ユーザー] アイコンをクリックします。 自分がアクセス許可を持っているテナントの一覧が表示されます。 管理するテナントを選択します。

![テナントの選択](media/choose-tenant.png "テナントの選択")
