# <a name="control-cloud-apps-with-policies"></a>ポリシーにるクラウド アプリの制御

ポリシーを使用すると、クラウド内のユーザーの動作を定義できます。 クラウド環境内の危険な動作、違反、疑わしいデータ ポイントやアクティビティを検出し、必要な場合には、修復ワークフローを統合してリスクを完全に抑制します。 収集するクラウド環境の情報の種類や実行する修復アクションの種類ごとに、さまざまな種類のポリシーが関連付けられています。  
  
たとえば、特定のデータ違反の脅威を検疫する場合と、組織内で使用されている危険なクラウド アプリをブロックする場合とでは、異なる種類のポリシーを設定する必要があります。  
  
## <a name="policy-types"></a>ポリシーの種類  
以下の種類のポリシーを作成できます。  
  
|ポリシーの種類|vmmblue_2|  
|-----------------|---------|  
|アクティビティ ポリシー|アクティビティ ポリシーでは、アプリ プロバイダーの API を利用してさまざまな自動プロセスを適用できます。 これらのポリシーを使用すると、さまざまなユーザーによって実行される特定のアクティビティを監視したり、特定の種類のアクティビティの割合が異常に高い場合に追跡したりできます。|  
|異常検出ポリシー|異常検出ポリシーでは、設定したリスク要因に基づいて、クラウドの異常な動作を監視し、組織のベースラインまたはユーザーの通常の動作と異なる動作を検出した場合に警告します。|  
|Cloud Discovery 異常検出ポリシー|Cloud Discovery 異常検出ポリシーでは、クラウド アプリの検出に使用するログを確認し、異常な動作を監視します。 たとえば、これまでに Dropbox を使用したことのないユーザーが突然 600 GB のファイルを Dropbox にアップロードしたり、特定のアプリで通常よりもはるかに多くのトランザクションが発生していたりする場合などに通知します。|  
|アプリ検出ポリシー|アプリ検出ポリシーでは、組織内で新しいアプリが検出された場合に通知するアラートを設定できます。|  
|[ファイル ポリシー]|ファイル ポリシーでは、クラウド アプリをスキャンし、指定したファイルまたはファイルの種類 (共有、外部ドメインとの共有)、データ (機密情報、個人情報、クレジット カード情報など) を監視して、ファイルにガバナンス アクションを適用します (ガバナンス アクションはクラウド アプリごとに異なります)。|  
  
## <a name="identifying-risk"></a>リスクの特定  
Cloud App Security では、クラウド内のさまざまなリスクを軽減することができます。 ポリシーやアラートを構成し、以下のリスクのいずれかに関連付けることができます。  
  
-   **アクセスの制御:** どのユーザーがどこから何にアクセスしているか。  
  
     継続的に動作を監視して、危険度の高い内部および外部からの攻撃などの異常なアクティビティを検出し、アプリまたはアプリ内の特定のアクションに対して警告、ブロック、本人確認を必要とするポリシーを適用します。 粒度の粗いブロックや粒度の細かい表示、編集、ブロックを使用した、ユーザー、デバイス、地理的情報に基づくオンプレミスおよびモバイル アクセス制御ポリシーを有効にします。 多要素認証の失敗、無効なアカウントからのログインの失敗、偽装イベントなど、疑わしいログイン イベントを検出します。  
  
-   **コンプライアンス:** コンプライアンス要件に違反していないか。  
  
     ファイル同期サービスに保存されている機密データや規制対象のデータ (各ファイルのアクセス許可の共有を含む) をカタログに追加して特定し、PCI、SOX、HIPAA などの規制を確実に遵守します。  
  
-   **構成の制御:** 構成が承認なく変更されていないか。  
  
     構成のリモート操作を含む、構成に対する変更を監視します。  
  
-   **Cloud Discovery:** 組織内で新規の承認されていないアプリが使用されていないか。 シャドウ IT (把握していないアプリが使用されている状態) の問題がないか。  
  
     規制遵守および業界の認定やベスト プラクティスに基づいて、各クラウド アプリの全般的なリスクを評価し、  
    ユーザー数、アクティビティ、トラフィック量、一般的な使用時間帯の監視が  
    各クラウド アプリケーションで行えます。  
  
-   **DLP:** 機密ファイルが一般公開されていないか。 ファイルを検疫する必要があるか。  
  
     オンプレミス DLP との統合により、既存のオンプレミス DLP ソリューションと統合してクローズドループ型の修復を提供します。  
  
-   **特権アカウント:** 管理者アカウントを監視する必要があるか。  
  
     特権のあるユーザーおよび管理者のアクティビティをリアルタイムで監視してレポートを作成します。  
  
-   **共有の制御:** クラウド環境内でデータがどのように共有されているか。  
  
     ファイルの内容とクラウドのコンテンツを検査して、内部および外部共有ポリシーを適用します。 共同作業を監視し、組織外でのファイル共有のブロックなどの共有ポリシーを適用します。  
  
-   **脅威の検出:** クラウド環境の脅威となる疑わしいアクティビティがないか。  
  
     ポリシー違反やアクティビティのしきい値に関するリアルタイム通知をテキスト メッセージまたは電子メールで受け取ります。 Cloud App Security では、機械学習アルゴリズムを適用することで、ユーザーがデータを悪用している可能性を示す動作を検出できます。  
  
## <a name="how-to-control-risk"></a>リスクを制御する方法  
ポリシーを使用してリスクを制御するには、以下の手順を実行します。  
  
1.  テンプレートまたはクエリからポリシーを作成します。  
  
2.  想定した結果を得るためにポリシーを調整します。  
  
3.  リスクに自動的に反応して修復する自動アクションを追加します。  
  
### <a name="create-a-policy"></a>ポリシーの作成  
すべてのポリシーの基盤として Cloud App Security のポリシー テンプレートを使用するか、クエリからポリシーを作成できます。  
  
ポリシー テンプレートを使用すると、環境内で特定のイベントを検出するために必要なフィルターと構成を適切に設定できます。 テンプレートにはすべての種類のポリシーが含まれ、さまざまなサービスに適用できます。  
  
**ポリシー テンプレート**からポリシーを作成するには、以下の手順を実行します。  
  
1.  コンソールで、[**制御**]、[**テンプレート**] の順にクリックします。  
  
     ![](./media/create-policy-from-template.png)  
  
2.  使用するテンプレートの行の右端にある **+** をクリックします。 定義済みのテンプレートの構成を含むポリシーの作成ページが開きます。  
  
3.  必要に応じてテンプレートを変更し、カスタム ポリシーを作成します。 この新しいテンプレートベースのポリシーのプロパティとフィールドは、すべて必要に応じて変更できます。  
> [!NOTE] 
>ポリシー フィルターを使用する場合、[**が次を含む**] を選択すると、コンマ、ピリオド、スペース、アンダースコアで区切られた完全に一致する単語のみが検索されます。 たとえば、**malware** または **virus** を検索する場合、virus_malware_file.exe は検出されますが、malwarevirusfile.exe は検出されません。 **malware.exe** を検索すると、ファイル名に malware または exe のいずれかを含むすべてのファイルが検出されますが、**"malware.exe"** (二重引用符を含む) を検索すると、malware.exe がそのまま名前に含まれるファイルのみが検出されます。 
[     **が次と等しい**] を使用して検索すると、完全な文字列のみが検索されます。たとえば、**malware.exe** を検索すると、malware.exe は検出されますが、malware.exe.txt は検出されません。  
4.  新しいテンプレートベースのポリシーを作成すると、ポリシーを作成したテンプレートの横にあるポリシー テンプレート テーブルの [**リンク済みポリシー**] 列に新しいポリシーへのリンクが表示されます。  
     各テンプレートからは任意の数のポリシーを作成できます。これらのポリシーはすべて元のテンプレートにリンクされるため、同じテンプレートを使用して構築されたすべてのポリシーを追跡することができます。  
  
また、**調査中にポリシーを作成する**こともできます。 **アクティビティ ログ**、**ファイル**、**アカウント**を調査し、ドリルダウンして特定の項目を確認する場合、調査の結果に基づいて新しいポリシーをいつでも作成することができます。  
  
たとえば、**アクティビティ ログ**を確認していて、ある管理者アカウントが想定外の地域からログインされていることを発見した場合、**アクティビティ ログ**の結果をフィルタリングして、その管理者によるアクティビティを記録したすべてのログを表示してから、そのユーザーのアクティビティが次に検出された場合に通知するレポートを作成できます。  
  
調査結果に基づいてポリシーを作成するには、以下の手順を実行します。  
  
1.  コンソールで、[**調査**] をクリックしてから [**アクティビティ ログ**]、[**ファイル**]、[**アカウント**] のいずれかをクリックします。  
  
2.  ページの上部にあるフィルターを使用して、疑わしい動作のみが表示されるように検索結果を絞り込みます。たとえば、[アクティビティ ログ] ページで、[**ユーザー**] をクリックして、アカウントに対して異常なアクティビティが記録された管理者を選択します。 次に、[**アクティビティ**] から [**フォルダーのコピー**] と [**ファイルのコピー**] を選択します。  
  
     ![](./media/create-file-from-investigation.png)  
  
3.  コンソールの右上にある [**検索に基づく新しいポリシー**] ![](./media/new-policy-from-search-button.png) をクリックします。  
  
4.  調査に使用したフィルターを含むポリシーの作成ページが開きます。  
  
5.  必要に応じてテンプレートを変更し、カスタム ポリシーを作成します。 この新しい調査ベースのポリシーのプロパティとフィールドは、すべて必要に応じて変更できます。  
   
> [!NOTE] 
> ポリシー フィルターを使用する場合、[**が次を含む**] を選択すると、コンマ、ピリオド、スペース、アンダースコアで区切られた完全に一致する単語のみが検索されます。 たとえば、**malware** または **virus** を検索する場合、virus_malware_file.exe は検出されますが、malwarevirusfile.exe は検出されません。  
[     **が次と等しい**] を使用して検索すると、完全な文字列のみが検索されます。たとえば、**malware.exe** を検索すると、malware.exe は検出されますが、malware.exe.txt は検出されません。

### <a name="policy-conflicts"></a>ポリシーの競合
複数のポリシーを作成した後に、ポリシーが重複する状況が発生することがあります。 その場合、Cloud App Security はポリシーを次のように処理します。
* あるポリシーのアクションに、別のポリシーのアクションが含まれている場合 (たとえば、**外部共有を削除する**は**非公開にする**に含まれます)、Cloud App Security は競合を解決して、強い方のアクションを適用します。
* アクションが完全に無関係である場合は (たとえば、**所有者に通知**と**非公開にする**)、 両方のアクションが実行されます。
* アクションが競合する場合 (たとえば、**所有者をユーザー A に変更する**と**所有者をユーザー B に変更する**)、一致ごとに結果が異なる可能性があります。 競合があるとドライブでの望ましくない変更を検出するのが困難な場合があるので、ポリシーを変更して競合を避けることが重要です。
