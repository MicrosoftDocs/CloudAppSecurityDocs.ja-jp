# <a name="control-cloud-apps-with-policies"></a>ポリシーによるクラウド アプリの制御

*適用対象: Microsoft Cloud App Security*

ポリシーを使用すると、ユーザーがクラウドで動作する方法を定義できます。 クラウド環境内の危険な動作、違反、疑わしいデータポイントやアクティビティを検出することができます。 必要に応じて、修復作業フローを統合して、リスクを完全に軽減することができます。 クラウド環境に関して収集するさまざまな種類の情報、および実行する必要のある修復アクションの種類に関連するポリシーには、複数の種類があります。

たとえば、検疫するデータ違反の脅威がある場合は、リスクの高いクラウドアプリが組織で使用されないようにする場合とは異なり、異なる種類のポリシーを設定する必要があります。

## <a name="identifying-risk"></a>リスクの特定

Cloud App Security は、クラウド内のさまざまなリスクを軽減するのに役立ちます。 次のいずれかのリスクに関連付けるポリシーとアラートを構成することができます。

- **アクセス制御:** だれがどこからアクセスできるか。

    動作を継続的に監視し、危険度の高い内部および外部の攻撃を含む異常なアクティビティを検出し、アプリまたはアプリ内の特定のアクションに対してアラートを出したり、ブロックしたり、id の検証を要求したりするポリシーを適用します。 粒度の粗いブロックと詳細な表示、編集、ブロックを使用して、ユーザー、デバイス、および地域に基づくオンプレミスおよびモバイルアクセス制御ポリシーを有効にします。 多要素認証の失敗、無効なアカウントログインの失敗、偽装イベントなど、疑わしいログインイベントを検出します。

- **コンプライアンス:** コンプライアンス要件が違反していないか。

    PCI、SOX、HIPAA などの規制に準拠するために、ファイル同期サービスに格納されている機密データや規制対象データ (各ファイルの共有アクセス許可を含む) をカタログ化および識別します。

- **構成コントロール:** 構成に対する承認されていない変更はありますか。

    リモート構成操作を含む構成の変更を監視します。

- **Cloud Discovery:** 組織で新しいアプリが使用されていますか? 不明なシャドウ IT アプリが使用されている問題が発生していますか。

    規制や業界の認定とベストプラクティスに基づいて、各クラウドアプリの全体的なリスクを評価します。 では、各クラウドアプリケーションのユーザー数、アクティビティ数、トラフィック量、一般的な使用時間を監視することができます。

- **DLP:** 専用ファイルは一般公開されていますか。 ファイルを検疫する必要がありますか。

    オンプレミスの DLP 統合によって、既存のオンプレミス DLP ソリューションと統合と終了ループの修復が可能になります。

- **特権アカウント:** 管理者アカウントを監視する必要がありますか。

    特権のあるユーザーと管理者のリアルタイムのアクティビティ監視とレポート。

- **共有コントロール:** クラウド環境でデータがどのように共有されているか。

    クラウド内のファイルとコンテンツの内容を検査し、内部および外部の共有ポリシーを適用します。 コラボレーションを監視し、組織外でのファイル共有のブロックなどの共有ポリシーを適用します。

- **脅威の検出:** クラウド環境に対して不審なアクティビティが発生していますか。

    テキストメッセージまたは電子メールを使用して、ポリシー違反やアクティビティのしきい値に関するリアルタイムの通知を受信します。 機械学習アルゴリズムを適用することにより、Cloud App Security は、ユーザーがデータを誤用していることを示す動作を検出できます。

## <a name="how-to-control-risk"></a>リスクを管理する方法

次の手順に従って、ポリシーによるリスクを制御します。

1. テンプレートまたはクエリからポリシーを作成します。

1. 予想される結果を得るために、ポリシーを微調整します。

1. 自動アクションを追加して、リスクに自動的に反応して修復します。

### <a name="create-a-policy"></a>ポリシーの作成

すべてのポリシーのベースとして Cloud App Security のポリシーテンプレートを使用するか、クエリからポリシーを作成することができます。

ポリシーテンプレートを使用すると、環境内で対象となる特定のイベントを検出するために必要なフィルターと構成を適切に設定できます。 テンプレートには、すべての種類のポリシーが含まれており、さまざまなサービスに適用できます。

ポリシー**テンプレート**からポリシーを作成するには、次の手順を実行します。

1. コンソールで、 **[コントロール]** 、 **[テンプレート]** の順にクリックします。

    ![テンプレートからポリシーを作成する](media/create-policy-from-template.png)

1. 使用するテンプレートの行の右端にある **+** をクリックします。 [ポリシーの作成] ページが開き、テンプレートの定義済みの構成が表示されます。

1. カスタムポリシーに合わせて、必要に応じてテンプレートを変更します。 この新しいテンプレートベースのポリシーのすべてのプロパティとフィールドは、必要に応じて変更できます。
   > [!NOTE]
   > ポリシーフィルターを使用する場合、には、完全な単語の検索だけ**が含まれ**ます (comas、ドット、スペース、またはアンダースコアで区切られています)。 たとえば、**マルウェア**または**ウイルス**を検索すると、virus_malware_file が検出されますが、malwarevirusfile.exe は見つかりません。 *マル*ウェアを検索すると、ファイル名にマルウェアまたは exe が含まれているすべてのファイルが検索されます。一方、 **"sysprep.exe"** を検索すると (二重引用符で囲まれている場合)、完全に "dism.exe" を含むファイルのみが検索されます。  
**Equals**は、完全な文字列のみを検索します。たとえば、*マル*ウェアを検索すると、マルウェアの .exe ではなく、マルウェアが検出されます。

1. 新しいテンプレートベースのポリシーを作成すると、ポリシーが作成されたテンプレートの横にあるポリシーテンプレートテーブルの [リンクされた**ポリシー** ] 列に新しいポリシーへのリンクが表示されます。
    各テンプレートから任意の数のポリシーを作成できます。これらはすべて元のテンプレートにリンクされます。 リンクを使用すると、同じテンプレートを使用して作成されたすべてのポリシーを追跡できます。

または、**調査中にポリシーを作成**することもできます。 **アクティビティログ**、**ファイル**、または**アカウント**を調査している場合は、調査の結果に基づいて新しいポリシーをいつでも作成することができます。

たとえば、**アクティビティログ**を参照していて、オフィスの IP アドレスの外部から管理者のアクティビティを確認している場合などです。

調査結果に基づいてポリシーを作成するには、次の手順を実行します。

1. コンソールで、 **[調査]** をクリックし、 **[アクティビティログ]** 、 **[ファイル]** 、または **[アカウント]** をクリックします。

1. ページの上部にあるフィルターを使用して、検索結果を疑わしい領域に限定します。 たとえば、アクティビティログ ページで、**アクティビティの種類** をクリックし、Azure 操作 の **管理者の作成** を選択します。 次に、 **[IP アドレス]** で **[カテゴリ]** を選択し、認識されたドメイン用に作成した ip アドレスカテゴリ (管理者、企業、VPN の ip アドレスなど) を含めないように値を設定します。

    ![調査からファイルを作成](media/create-file-from-investigation.png)

1. コンソールの右上隅にある **[検索から新しいポリシーを作成]** をクリックします。

    ![[検索] ボタンの新しいポリシー](media/new-policy-from-search-button.png)

1. [ポリシーの作成] ページが開き、調査で使用したフィルターが表示されます。

1. カスタムポリシーに合わせて、必要に応じてテンプレートを変更します。 この新しい調査ベースのポリシーのすべてのプロパティとフィールドは、必要に応じて変更できます。

    > [!NOTE]
    > ポリシーフィルターを使用する場合、には、完全な単語の検索だけ**が含まれ**ます (comas、ドット、スペース、またはアンダースコアで区切られています)。 たとえば、**マルウェア**または**ウイルス**を検索すると、virus_malware_file が検出されますが、malwarevirusfile.exe は見つかりません。  
**Equals**は、完全な文字列のみを検索します。たとえば、**マル**ウェアを検索すると、マルウェアの .exe ではなく、マルウェアが検出されます。

    ![調査からアクティビティポリシーを作成する](media/create-activity-policy-from-investigation.png)

    > [!NOTE]
    > ポリシーフィールドの設定の詳細については、対応するポリシーのドキュメントを参照してください。

### <a name="add-automated-actions-to-respond-and-remediate-risks-automatically"></a>自動アクションを追加して、リスクに自動的に反応して修復する

アプリあたりの使用可能なガバナンスアクションの一覧については、「[管理対象アプリ](governance-actions.md)」を参照してください。

また、一致が検出されたときに電子メールまたはテキストメッセージでアラートを送信するようにポリシーを設定することもできます。

通知の設定を設定するには、[ポータルをカスタマイズ](general-setup.md)します。

> [!NOTE]
> テキストメッセージで送信されるアラートの最大数は、1日あたり10件の電話番号です。 日は UTC タイムゾーンに従って計算されます。

## <a name="enable-and-disable-policies"></a>ポリシーを有効または無効にする

作成したポリシーは、有効または無効にすることができます。 を無効にすると、ポリシーを停止するために、ポリシーを作成した後に削除する必要がなくなります。 その代わりに、何らかの理由でポリシーを停止する場合は、再度有効にするまで無効にします。

- ポリシーを有効にするには、 **[ポリシー]** ページで、有効にするポリシーの行の最後にある3つの点をクリックします。 **[有効]** を選択します。

    ![ポリシーを有効にする](media/enable-policy.png)

- ポリシーを無効にするには、 **[ポリシー]** ページで、無効にするポリシーの行の最後にある3つの点をクリックします。 **[無効]** を選択します。

    ![ポリシーを無効にする](media/disable-policy.png)

既定では、新しいポリシーを作成すると有効になります。

## <a name="policies-overview-report"></a>ポリシーの概要レポート

Cloud App Security を使用すると、ポリシーごとに集計されたアラートメトリックを示すポリシーの概要レポートをエクスポートして、組織をより効果的に保護するためのポリシーの監視、理解、およびカスタマイズを行うことができます。

ログをエクスポートするには、次の手順を実行します。

1. **[ポリシー]** ページで、 **[エクスポート]** ボタンをクリックします。

1. 必要な時間の範囲を指定します。

1. **[エクスポート]** をクリックします。 この処理には時間がかかることがあります。

エクスポートされたレポートをダウンロードするには:

1. レポートの準備ができたら、 **[設定]** 、エクスポートされた **[レポート]** の順に選択します。

1. テーブルで、ポリシーの一覧の**概要レポート**から関連するレポートを選択し、[ダウンロード] をクリックします。

    ![[ダウンロード] ボタン](media/download-button.png)
