# <a name="anomaly-detection-policy"></a>異常検出ポリシー
この記事は、ポリシーに関する参照の詳細です。各ポリシーの種類、およびポリシーごとに設定できるフィールドについて詳しく説明します。  

組織が Cloud App Security を導入してセキュリティ確保すると、すべてのクラウド アクティビティには、事前定義されたさまざまなリスク要因に基づいてスコアが付けられます。 Cloud App Security によってクラウド上のすべてのユーザー セッションが監視され、ここで設定したリスク要因を考慮して、組織の基準またはユーザーの通常アクティビティとは異なる状況が発生した場合にアラート通知されます。 ポリシーは、さまざまなユーザー、場所、および組織部門に合わせて異なる方法で適用できます。 たとえば、社内の IT チームのメンバーが、社外から頻繁にアクセスしているときにアラート通知するポリシーを作成できます。  

Cloud App Security には、7 日間の初回学習期間があります。この間はどのような新しいユーザー、アクティビティ、デバイス、場所にも異常フラグは設定されません。 この期間が過ぎると、過去に検出されたアクティビティ、ユーザーがアクティブになっていた時間、IP アドレス、デバイスと各セッションが比較され、また、そのようなアクティビティのリスク スコアと各セッションが比較されます。 


次の異常検出ポリシーを使用できます。

**あり得ない移動**
- この検出は、ユーザーの環境をプロファイリングし、組織のアクティビティの学習したベースラインから警告をトリガーするヒューリスティック異常検出エンジンの一部です。 これは、地理的に離れた場所からの 2 回のユーザー アクティビティ (1 つまたは複数のセッション) が、最初の場所から 2 番目の場所に移動するのにかかると予想される時間よりも短い期間に発生する場合、別のユーザーが同じ資格情報を使用しているものとして識別します。 この検出では、あり得ない移動で使用される、組織内の他のユーザーが普通に使用している VPN や場所などの明らかな "誤検知" を無視する機械学習アルゴリズムが使用されています。 この検出には、7 日間の初期学習期間があり、その期間に新しいユーザーのアクティビティ パターンが学習されます。


**頻度の低い国からのアクティビティ**
- この検出は、ユーザーの環境をプロファイリングし、組織のアクティビティの学習したベースラインから警告をトリガーするヒューリスティック異常検出エンジンの一部です。 この検出では、過去にアクティビティがあった場所を考慮し、新規の場所とあまり使用されない場所が判別されます。 この異常検出エンジンでは、組織のユーザーが以前に使用した場所の情報が保持されます。 組織のあるユーザーや任意のユーザーが、最近使用されていない場所や使用されたことがない場所からアクティビティを発生させると、アラートがトリガーされます。 この検出には、7 日間の初期学習期間があり、その期間はどの新しい場所に対してもアラートはトリガーされません。


**匿名 IP アドレスからのアクティビティ**
- この検出は、ユーザーの環境をプロファイリングし、組織のアクティビティの学習したベースラインから警告をトリガーするヒューリスティック異常検出エンジンの一部です。 この検出では、匿名のプロキシ IP アドレスと認識されている IP アドレスからユーザーがアクティブであったことが識別されます。 これらのプロキシは、デバイスの IP アドレスを隠したいユーザーによって使用されますが、悪意のある目的で使用されることもあります。 この検出には、組織内のユーザーに広く使用されている誤ってタグ付けされた IP アドレスなどの "誤検知" を減らす機械学習アルゴリズムが使用されています。

**不審な IP アドレスからのアクティビティ**
- この検出は、ユーザーの環境をプロファイリングし、組織のアクティビティの学習したベースラインから警告をトリガーするヒューリスティック異常検出エンジンの一部です。 この検出では、Microsoft 脅威インテリジェンスによってリスクがあると判断された IP アドレスからユーザーがアクティブであったことが識別されます。 これらの IP は、ボットネット C & C などの悪意のあるアクティビティに関係する、侵害されたアカウントである可能性があります。 この検出には、組織内のユーザーに広く使用されている誤ってタグ付けされた IP アドレスなどの "誤検知" を減らす機械学習アルゴリズムが使用されています。


**(ユーザーによる) 異常な X アクティビティ**
- この検出は、ユーザーの環境をプロファイリングし、組織のアクティビティの学習したベースラインから警告をトリガーするヒューリスティック異常検出エンジンの一部です。 この検出は、1 つのセッションで複数回 X アクティビティを実行し、学習したベースラインに関してセキュリティ違反を試みている可能性のあるユーザーを識別します。 この検出は、ユーザーのログオン パターンをプロファイリングする機械学習アルゴリズムを使用しており、"誤検知" を減らします。


**複数回失敗したログイン試行**
- この検出は、ユーザーの環境をプロファイリングし、組織のアクティビティの学習したベースラインから警告をトリガーするヒューリスティック異常検出エンジンの一部です。 この検出は、1 つのセッションで複数回ログインを試行して失敗した、学習したベースラインからセキュリティ違反を試みている可能性のあるユーザーを識別します。 この検出は、ユーザーのログオン パターンをプロファイリングする機械学習アルゴリズムを使用しており、"誤検知" を減らします。

異常検出ポリシーを設定するには:  
  
1.  コンソールで、**[制御]**、**[ポリシー]** の順にクリックします。  
  
2.  **[ポリシーの作成]** をクリックしてから **[異常検出]** ポリシーを選択します。  
  
     ![異常検出ポリシー メニュー](./media/anomaly-detection-policy-menu.png "Anomaly detection policy menu")  
  
3.  ポリシーの名前と説明を入力してから、**[アクティビティ フィルター]** フィールドでポリシーを適用するアクティビティを選択します。  
  
4.  ポリシーの名前と説明を入力します。この場合、ポリシー テンプレートを使用することもできます。ポリシー テンプレートの詳細については、「[Control cloud apps with policies (ポリシーによるクラウド アプリの制御)](control-cloud-apps-with-policies.md)」を参照してください。  
  
5.  クラウド環境内のすべてのアクティビティにポリシーを適用するには、**[すべての監視対象アクティビティ]** を選択します。 特定種類のアクティビティにポリシーを制限するには、**[選んだアクティビティ]** を選択します。 **[フィルターの追加]** をクリックし、アクティビティをフィルターするための適切なパラメーターを設定します。 たとえば、Salesforce の管理者によって実行されるアクティビティのみにポリシーが適用されるようにするには、このユーザー タグを選択します。  
  
6.  このフィールドの下で **[リスク要因]** を設定します。 リスク スコアを計算する際に考慮するリスク ファミリを選択できます。 行の右側にある [オン/オフ] ボタンを使用すると、各種のリスクを有効または無効にできます。 さらに詳細に設定するには、特定のリスクファミリそれぞれで有効にするアクティビティを選択します。  
  
     リスク要因は次のとおりです。  
  
    -   **ログインの失敗**: ユーザーがログインを試みているが、短時間で複数回失敗している場合。  
  
    -   **管理者アクティビティ**: 管理者が特権アカウントを使用して、通常とは異なる場所または時間にログインしている場合。  
  
    -   **非アクティブなアカウント**: しばらく使用されていなかったアカウントが突然アクティブになった場合。  
  
    -   **場所**: 通常とは異なる、疑わしいまたはこれまでにない場所でアクティビティが実行された場合。  
  
    -   **不可能な移動**: ユーザーがデンバーでログインしてから、10 分後にパリからログインしている場合。  
  
    -   **デバイスとユーザー エージェント**: 認識できない、または管理されていないデバイスからアクティビティが実行された場合。  

    -   **アクティビティ レート**: 特定のアプリで急激にアクティビティが増えている場合。 大量のダウンロードや削除、大量のファイル共有、予期しない大量の管理アクティビティが実行された場合。
  
     これらのパラメーターを使用して複雑なシナリオを定義できます。たとえば、異常検出の対象となるリスク要因からオフィスの IP 範囲を除外するには、特定の office IP タグを作成し、その IP 範囲をフィルターにかけて対象パラメーターから除外します。 次に、以下の手順に従って、作成した範囲を管理アクティビティの異常検出から除外します。  
  
    -   **[リスクの種類]** で **[管理者アクティビティ]** を見つけます。  
  
    -   **[適用先]** を **[選択されたアクティビティ]** に変更します。  
  
    -   **[アクティビティ フィルター]** で、**[適用先]** を **[選択されたアクティビティ]** に設定して、 **[Activities matching all of the following (以下のすべてに一致するアクティビティ) ]** で **[管理者アクティビティ]** を **[True]** に設定します。  
  
    -   **+** アイコンをクリックして、**[IP tag does not equal (IP タグが等しくない)]** を選択してから先ほどの Office IP タグを選択します。  
  
7.  **[感度]** でアラートの受信頻度を選択します。  
  
     感度値で、1 週間あたり 1,000 ユーザーごとにトリガーされる平均アラート数が決まります。  
  
     ![異常検出 IP](./media/anomaly-detection-ips.png "anomaly detection IPs")  
  
8.  **[作成]** をクリックします。  
 

## <a name="anomaly-detection-policy-reference"></a>異常検出ポリシーの参照  
異常検出ポリシーでは、異常な動作についてユーザーのアクティビティを継続的に監視する機能をセットアップし構成することができます。 異常はユーザーのアクティビティをスキャンして検出します。 リスクは、6 つのリスク要因にグループ化されている 30 種類を超えるさまざまなリスク インジケーターを使用して評価されます。 ポリシーの結果に基づいて、セキュリティ アラートがトリガーされます。   
各ポリシーは、次の部分で構成されます。  
  
-   アクティビティ フィルター - フィルタリングされたユーザー アクティビティの異常のみを選択的にスキャンします。  
  
-   リスク要因 - リスクを評価するときに使用するリスク要因を選択できます。  
  
-   感度 - アラートによりポリシーがトリガーされるまでの回数を設定できます。  
  
### <a name="activity-filters"></a>アクティビティ フィルター  
アクティビティ フィルターの一覧については、「[ここにリンクの説明を入力](activity-filters.md)」を参照してください。  
  
### <a name="risk-factors"></a>リスク要因  
以下に、ユーザー アクティビティのリスクを評価する際に考慮されるリスク要因のリストを示します。 それぞれのリスク要因は、オンとオフを切り替えることができます。 各リスク要素については **[Apply to (適用)]** フィールドで 2 つのオプションを選択可能で、ユーザー アクティビティのリスクを評価するときにこれを含めるかどうかを決定することができます。  
  
-   すべての監視対象アクティビティ - ポリシーのアクティビティ フィルターを通過するユーザー アクティビティをすべて含めます。  
  
-   選択されたアクティビティ - ポリシーのアクティビティ フィルターとこのリスク要因で表示されるアクティビティ フィルターの両方を通過するユーザー アクティビティを含めます。 このオプションを選択するとリスク要因でアクティビティ フィルター セレクターが表示されます。これは、ポリシーのアクティビティ フィルターとまったく同じように働きます。  
  
それぞれのリスク要因について、リスク評価に含まれていると評価対象のユーザー アクティビティのリスクが増加する原因となる独自のトリガーがあります。  
  
-   ログインの失敗 - ログインの失敗回数が多い、または失敗したログインのみで構成されたアクティビティ。  
  
-   管理アクティビティ - 予期しないユーザーによって実行された、または新規か組織の他のユーザーが使用していない IP、ISP、場所で実行された管理アクティビティ。  
  
-   非アクティブなアカウント - 長期間にわたってアクティブでないユーザーによって実行されたアクティビティ。  
  
-   場所 - 他のユーザーまたはその特定のユーザーが過去に一度も使用していないか、まったく使用されたことがない、あるいは過去にログインの失敗のみが記録されている IP、ISP、場所で実行されたアクティビティ。  
  
-   不可能な移動 - 短時間で離れた場所から実行されたアクティビティ。  
  
-   デバイスとユーザー エージェント - 他のユーザーまたはその特定のユーザーが過去に一度も使用していない、あるいはまったく使用されたことがないユーザー エージェントやデバイスで実行されたアクティビティ。  
  
-   アクティビティ レート - 短時間でユーザーが繰り返し実行するアクティビティ。 

### <a name="sensitivity"></a>秘密度  
このポリシーによってトリガーされるアラート数を制御するには、**[日次アラート制限]** を設定し、1 日に発生するアラート数を制限します。
