# <a name="daily-activities-to-protect-your-cloud-environment"></a>クラウド環境を保護するための日常的な作業
Cloud App Security をセットアップして実行を開始し、データ ストリームを構成して、ユーザーに任意のアプリの使用を許可し、クラウド環境を監視するポリシーを設定したら、Cloud App Security を使用してクラウドの保護や制御を行いリスクを管理します。  
  
このトピックでは、日常的に行う必要がある作業について説明します。  
  
## <a name="check-the-dashboard"></a>ダッシュボードの確認  
Cloud App Security ポータルを開くと、自社クラウド環境で発生中のアラート、アクティビティの違反、コンテンツの違反、地理的にどの場所のユーザーがアクティビティを実行したかがプロットされたアクティビティ マップ、接続中のアプリの使用状況の傾向が示されます。  
  
日常的にダッシュ ボードを見てどのようなアラートが新たにトリガーされているかを確認し、それに対応することをお勧めします。 これは、クラウド環境全体で何が起こっているかを大まかに把握し、クラウド環境の正常性を監視するうえでも役立ちます。  
  
![ダッシュボード](./media/dashboard.png "dashboard")  
  
## <a name="handle-your-alerts"></a>アラートの処理  
アラートは、クラウド環境をより深く理解するための最初のステップです。 確認した状況に基づいて新しいポリシーを作成できます。 たとえば、グリーンランドからログオンした管理者を発見した場合、今後グリーンランドからログオンした場合に管理者アカウントを自動的に停止するポリシーを作成できます。  
  
すべてのアラートを確認し、これに基づいてポリシーを修正することをお勧めします。 無害なイベントが既存のポリシーに違反していると判断される場合、ポリシーを調整して不要なアラートを受信する回数を減らします。  
  
-   [**アラートを開く**] を選択して [**すべてのアラートを表示**] をクリックします。  
  
     この画面では不審なアクティビティや作成済みのポリシーの違反がすべて表示されるので、クラウド環境で定義されているセキュリティ体制の保護に活用できます。  
  
     ![アラート](./media/alerts.png "alerts")  
  
-   それぞれのアラートについて、違反の内容を調査し必要な対応を決定する必要があります。  
  
     アラートは、**アラートの種類**や**重要度**でフィルタリングすると最も重要なものから対応できます。  
  
     いずれかのアラートをクリックします。 アラートの種類に応じて、アラートを解決するためのさまざまな対応方法が表示されます。  
  
     アラートの調査について、対応が必要な違反には 3 つの種類があります。  
  
    -   即座に対応が必要な**重大な違反**  
  
         例:  
  
         不審なアクティビティのアラートの場合、ユーザーがパスワードを変更するまでアカウントを停止することができます。  
  
         データが漏えいした場合は、アクセス許可を制限するか、ファイルを検疫することができます。  
  
         許可されていない新規のサービスが発見された場合、プロキシまたはファイアウォールでサービスへのアクセスをブロックできます。  
  
    -   さらに詳細な調査が必要な**問題のある違反**  
  
         アクティビティの内容について、ユーザーまたはユーザーのマネージャーに連絡することができます。  
  
         詳細を把握するまでは、アクティビティをそのままにしておきます。  
  
    -   正当な使用によって発生した**承認された違反または変則的な動作**  
  
         アラートを消去します。  
  
-   このプロセスが完了したら、アラートを解決済みとしてマークします。  
  
次の表は、トリガーされるアラートの種類と推奨される解決方法の一覧です。  
  
|アラートの種類|説明|推奨される解決方法|  
|----------------|-----------------|----------------------------|  
|アクティビティ ポリシー違反|この種類のアラートは、作成されたポリシーによるものです。|- この種類のアラートが大量に発生する場合は、ポリシー センターで直接操作して発生数を減らすことをお勧めします。<br />- フィルターを追加してより細かく制御し、不要なエンティティを除外するようにポリシーを調整します。<br />- ポリシーが非常に正確でアラートが正当なものであり、違反が発生しないようにすぐに変更したい場合は、ポリシーに自動修復を追加して自動応答させることを検討します。|  
|ファイル ポリシー違反|この種類のアラートは、作成されたポリシーによるものです。|- この種類のアラートが大量に発生する場合は、ポリシー センターで直接操作して発生数を減らすことをお勧めします。<br />- フィルターを追加してより細かく制御し、不要なエンティティを除外するようにポリシーを調整します。<br />- ポリシーが非常に正確でアラートが正当なものであり、違反が発生しないようにすぐに変更したい場合は、ポリシーに自動修復を追加して自動応答させることを検討します。|  
|危険な状態のアカウント|Cloud App Security が危険な状態のアカウントを検出した場合 (承認されていない方法でアカウントが使用された可能性がとても高い場合) にこの種類のアラートがトリガーされます。|ユーザーに連絡して、該当するユーザーがパスワードを変更したことを確認するまでアカウントを停止することをお勧めします。|  
|非アクティブなアカウント|接続中のクラウド アプリのいずれかで使用されていないアカウントが存在する場合にこのアラートがトリガーされます。|該当するアカウントが使用されているかどうかをユーザーおよびユーザーのマネージャーに確認します。 使用されていない場合は、ユーザーの使用を一時的に停止し、アプリのライセンスを解除します。|  
|新しい管理ユーザー|このアラートは、接続中のアプリケーションで特権アカウントが変更された場合にトリガーされます。|新しい管理者権限が実際にユーザーから要求されていることを確認し、要求が確認されない場合は管理者権限を取り消してリスクを軽減することをお勧めします。|  
|新しい場所の管理者|このアラートは、接続中のアプリケーションで特権アカウントが変更された場合にトリガーされます。|変則的な場所からのログオンが正当であることを確認し、正当ではない場合は、管理者権限を取り消すか、アカウントを一時的に停止してリスクを軽減することをお勧めします。|  
|新しい場所|これは接続中のクラウド アプリに新しい場所からアクセスされたことを示すアラートで、1 か国につき 1 回のみ発生します。|該当するユーザーのアクティビティを調査します。|  
|新しいサービスの検出|これはシャドウ IT に関するアラートで、Cloud Discovery が新しいアプリを検出した場合にトリガーされます。|<ul><li>アプリ カタログに基づいてサービスのリスクを評価します。</li><li>アクティビティをドリル ダウンして使用パターンと普及状態を把握します。</li><li>アプリの使用を許可するか、許可しないかを決定します。<br /><br />     使用が許可されていないアプリの場合:<br /><br /> <ul><li>プロキシまたはファイアウォールで使用をブロックできます。</li><li>同じカテゴリ内に使用が許可されているアプリと許可されていないアプリが同時に存在する場合、ドリルダウンして使用が許可されていないアプリのユーザーのリストをエクスポートし、許可されているアプリのリストに移行することができます。</li></ul></li></ul>|  
|不審なアクティビティ|このアラートは、社内のユーザーが予期されるアクティビティとは異なる変則的なアクティビティを行ったことが検出されるとトリガーされます。|動作を調査し、ユーザーに確認します。<br /><br /> この種類のアラートは、自社環境について詳しく把握し、新しいポリシーを作成するきっかけとして適しています。 たとえば、任意のユーザーが接続中のアプリに突然大量のデータをアップロードした場合、このような変則的な動作を制御するルールを設定します。|  
|クラウドの不審な使用|このアラートは、社内のユーザーが予期されるアクティビティとは異なる変則的なアクティビティを行ったことが検出されるとトリガーされます。|動作を調査し、ユーザーに確認します。<br /><br /> この種類のアラートは、自社環境について詳しく把握し、新しいポリシーを作成するきっかけとして適しています。 たとえば、任意のユーザーが接続中のアプリに突然大量のデータをアップロードした場合、このような変則的な動作を制御するルールを設定します。|  
|個人アカウントの使用|このアラートは、新しい個人アカウントが接続中のアプリのリソースへのアクセス許可を取得するとトリガーされます。|外部アカウントでは、ユーザーの共同作業者を削除します。|  
  
## <a name="use-policies-to-assess-risk"></a>ポリシーを使用してリスクを評価  
発生中のアラートを確認した後は、**ポリシー センター**に移動して、アラートをトリガーしないポリシー違反を確認します。  
  
-   Cloud App Security ポータルで [**制御**]、[**ポリシー**] の順にクリックします。  
  
-   特定のポリシーをクリックして、アラートをトリガーしていないポリシーの一致の [**Violating now (発生中の違反)**] リストを確認します。  
  
-   違反を 1 つずつクリックし、それぞれの処理を決定します。 ガバナンス アクションの詳細については、以下を参照してください。  
  
     たとえば、コンプライアンス違反を発見するようにポリシーが設定されていて、クレジット カード番号のファイルが OneDrive に保存された場合、ポリシーに一致します。  
  
     ![pci 一致](./media/pci-matches.png "pci matches")  
  
-   一致をクリックして、ポリシーに違反しているファイルを参照します。  
  
     ![pci コンテンツ一致](./media/pci-content-matches.png "pci content matches")  
  
     ファイル自体をクリックすると、そのファイルに関する情報を取得できます。  
  
     [**グループ作業者**] をクリックすると、そのファイルにアクセスできるユーザーが表示されます。  
  
     [**一致**] をクリックすると、実際のクレジット カード番号が表示されます。  
  
     ![コンテンツ一致 ccn](./media/content-matches-ccn.png "content matches ccn")
