# <a name="protecting-your-organization-against-threats"></a>脅威から組織を保護する  

危険性の高い使用とクラウドのセキュリティに関する問題を特定し、異常なユーザー動作を検出して、接続しているクラウド アプリでの脅威を防ぎます。 ユーザーと管理者のアクティビティを把握し、疑わしい動作が見られる場合や、危険と思われる特定のアクティビティが承認された環境で発生した場合に自動的にアラートを生成するポリシーを定義します。 膨大な量の Microsoft 脅威インテリジェンスおよびセキュリティ調査データから情報を取り出します。 脅威検出ポリシーは、承認されたアプリで必要なすべてのセキュリティ制御が確実に実行されるようにするのに役立ちます。また、その制御を維持するのにも役立ちます。
 
## <a name="mass-download-by-a-single-user-data-exfiltration-by-an-insider"></a>単独のユーザーによる大量ダウンロード (内部者によるデータの持ち出し)

このユース ケースは、Office 365、G Suite、Box、Dropbox、Salesforce に適用されます。

### <a name="the-threat"></a>脅威の内容
悪意のある内部者は、複数のファイルのダウンロードまたはアクセスにより、短期間で Office 365 や他のクラウド アプリからデータを抜き取ることができます。

### <a name="the-solution"></a>解決策
ユーザーが短期間に大量のファイルのダウンロードまたはアクセスを行うタイミングを検出します。

#### <a name="prerequisites"></a>前提条件

Cloud App Security に、少なくとも 1 つのクラウド アプリを[接続](enable-instant-visibility-protection-and-governance-actions-for-your-apps.md)します。

#### <a name="setting-up-monitoring"></a>監視の設定

1.  既定では、Cloud App Security はネットワークをスキャンして基準を確立し、クラウドでユーザーが通常行う動作、タイミング、よく行う動作のパターンを学習します。 

2. クラウド アプリの大量ダウンロードを監視し、通常とは異なる状態が生じた場合にアラートを生成するポリシーを設定します。

    1. **[ポリシー]** ページで、[**[アクティビティ ポリシーの作成]**](user-activity-policies.md) をクリックします。 
   

    2. [**[ポリシー テンプレート]**](policy-template-reference.md) フィールドで、**[1 人のユーザーによる大量ダウンロード]** を選択します。
    
    3. 必要に応じて、反復アクティビティ フィルターを微調整できます。
    
    4. [**作成**] をクリックします。 
   
     
2. 一致を調査する
    
    1. **[ポリシー]** ページで、ポリシー名をクリックして **[ポリシー レポート]** に移動し、そのポリシーにトリガーされた一致を確認します。

    2. 特定の一致をクリックしてアクティビティ ドロワーを開き、一致を調査することができます。 ドロワーでは、このアクティビティと一致した他のポリシーを確認できます。 
    
    3. ユーザーがダウンロードしたファイルを確認するには、**アクティビティ ドロワー**でアクティビティ オブジェクトをクリックし、ファイル名をクリックします。ファイル一覧のファイルが表示されます。 ここで、ファイルがそのユーザーに所有されているかどうか、誰と共有されているかを確認できます。また、それらのユーザーが通常使用しているファイルかどうか、ダウンロードすべきではない内部 IP かどうかを判断できます。
     


#### <a name="validating-your-policy"></a>ポリシーの検証

1. アラートをシミュレートするには、接続しているクラウド アプリから短時間で、設定したしきい値よりも多数のドキュメントをダウンロードします。"短時間" は、ポリシーで設定した時間によって異なります (たとえば、5 分間未満で 30 回のダウンロードなど)。
3. ポリシー レポートに移動します。 アクティビティ ポリシーの一致がすぐに表示されます。 
4. この一致をクリックして、ダウンロードされたファイルを確認することができます。  

#### <a name="removing-the-risk"></a>リスクの排除

検証が完了し、ポリシーを微調整したら、ポリシーと一致した可能性のある誤検出を削除します。たとえば、ファイル、会社イベントの写真のフォルダーなどを同期しているサービス アカウントです。次に、以下の操作を行います。 
  1. アクティビティ ドロワーでアクティビティを開いて[ガバナンス アクション](governance-actions.md)を実行できます。次に、**[アクティビティ オブジェクト]** のファイル名をクリックします。

  2. ユーザーに連絡して、ユーザーのコンピューターでそれほど多数の会社ファイルをコピーする必要がある理由を確認します。

 
## <a name="admin-activity-from-outside-your-organizations-network"></a>組織のネットワーク以外からの管理者アクティビティ

このユース ケースは、Office 365、G Suite、Box、Dropbox、Salesforce、Okta に適用されます。

### <a name="the-threat"></a>脅威の内容
外部の攻撃者により管理者アカウントが侵害された場合を仮定します。 管理者アカウントは、最高の特権を持ち組織内のすべての情報にアクセスできるので、組織で最も機密性の高いアカウントです。 通常、管理者アクティビティは、管理者の仕事の一環としてオフィス内から実行する必要があります。リモートの場所や認識されていないデバイスからは実行しません。

### <a name="the-solution"></a>解決策
管理者を装ったユーザーが外部の IP アドレスからクラウド アプリケーションに接続し、何らかの管理者アクティビティを実行しているときに検出します。

#### <a name="prerequisites"></a>前提条件

- Cloud App Security に、少なくとも 1 つのクラウド アプリを[接続](enable-instant-visibility-protection-and-governance-actions-for-your-apps.md)します。

- **[設定]** > **[IP アドレスの範囲]** に移動し、内部サブネットとそのエグレス パブリック IP の両方について IP アドレスの範囲を追加し、**企業**とタグを付けます。

#### <a name="setting-up-monitoring"></a>監視の設定

1.  クラウド アプリの監視を開始するには、クラウド アプリのネットワーク外からの管理者アクティビティを監視し、通常とは異なる動作が行われている場合にアラートを生成するポリシーを設定します。

    1. **[ポリシー]** ページで、[**[アクティビティ ポリシーの作成]**](user-activity-policies.md) をクリックします。 
   

    2. [**[ポリシー テンプレート]**](policy-template-reference.md) フィールドで **[企業 IP アドレス以外からの管理アクティビティ]** を選択し、**[アクティビティの種類]** も **[ログオン]** に設定します。また、**[ユーザー]**、**[グループから]** の順に選択し、管理者が属するグループを選択します。
    
    3. ポリシーを微調整して、疑わしいと見なされるアクティビティの反復回数を設定できます。
    
    4. [**作成**] をクリックします。 
   
     
2. 一致を調査する
    
    1. **[ポリシー]** ページで、ポリシー名をクリックして **[ポリシー レポート]** に移動し、そのポリシーにトリガーされた一致を確認します。

    2. 特定の一致をクリックしてアクティビティ ドロワーを開き、一致を調査することができます。 ドロワーでは、このアクティビティと一致した他のポリシーを確認できます。 
     
#### <a name="validating-your-policy"></a>ポリシーの検証

1. アラートをシミュレートするには、企業ネットワークではない IP アドレスを使用するコンピューターから、設定したアクティビティ回数のしきい値を超えるように、管理者アクティビティを短時間で実行します。"短時間" は、ポリシーで設定した時間によって異なります (たとえば、5 分間未満で 1 回のアクティビティなど)。
3. ポリシー レポートに移動します。 アクティビティ ポリシーの一致がすぐに表示されます。 
4. 一致をクリックすると、実行されたアクティビティを確認できます。 

#### <a name="removing-the-risk"></a>リスクの排除

リスクを検証してポリシーを微調整したら、ポリシーと一致する可能性があった誤検知を排除します。 次に、以下の操作を行います。 
  1. アクティビティ ドロワーでアクティビティを開いて[ガバナンス アクション](governance-actions.md)を実行できます。次に、**[ユーザー]** の名前をクリックします。

  2. 表示されるユーザー ページには、過去 1 か月間のユーザー アクティビティのグラフと、過去 1 か月間でユーザーがアクティブになった場所、グループ メンバーシップ、アプリ アクティビティ、およびアカウントが表示されます。 
  
  3. 必要に応じて、**連絡先**をクリックしてユーザーに電子メール メッセージを送信し、アカウントが侵害されたことを警告することができます。

 ![外部自動ガバナンス](./media/auto-gov-external.png)

   2. 検証が完了したら、自動ガバナンス アクションを実行するように設定できます。 たとえば、**[ユーザーの停止]**、**[ユーザーのコラボレーションを削除]** などです。
